## 3MF Consortium Integration
- **lib3mf WASM** (`@3mfconsortium/lib3mf` v2.5.0) for spec-compliant 3MF file parsing on the server
- **3mfViewer** from [3MFConsortium](https://github.com/3MFConsortium/3mfViewer) embedded as an iframe viewer ŌĆö full scene tree, materials, wireframe, beam lattice support
- Spec-compliant metadata, thumbnail, and mesh extraction from 3MF files
- 3MF file validation endpoint (`POST /api/library/validate`)

## Gcode Toolpath Viewer
- Server-side gcode parser extracts G0/G1 toolpath from any gcode file
- Per-layer colour gradient visualisation (blue bottom ŌåÆ red top)
- Automatic downsampling for large files (max 100k segments)
- Gzipped caching in `data/toolpath-cache/` for instant repeat views
- Works for **all printer types** ŌĆö Moonraker HTTP API and Bambu FTPS

## Three.js Enhanced Viewer
- **Three.js r183** vendored (no CDN dependency) with OrbitControls
- Smooth shading, 3-point lighting, build plate grid
- Print progress clipping plane with green edge glow
- Per-layer vertex colours for multi-colour print tracking
- Auto-rotation with user interaction detection

## Universal 3D Preview
- `/api/preview-3d` endpoint resolves 3MF from multiple sources: library, slicer uploads, history, printer FTPS
- Automatic strategy selection per printer type via `printer-capabilities.js`
- Bambu Lab: FTPS download with fuzzy name matching across `/`, `/cache/`, `/sdcard/`
- Moonraker/Klipper: HTTP download from `/server/files/gcodes/`
- Falls back to gcode toolpath when no 3MF mesh is available
- Extracts embedded gcode from Bambu `gcode.3mf` files (which contain no mesh data)

## 3D Preview Everywhere
- **History** ŌĆö 3D button with upload/replace/delete for linked 3MF files
- **Library** ŌĆö 3mfViewer for all 3MF files with full material rendering
- **Queue** ŌĆö 3D button on queued 3MF files
- **Scheduler** ŌĆö 3D button on scheduled print events
- **Gallery** ŌĆö 3D button on completed print gallery items

## History 3MF Linking
- Upload 3MF files directly to history entries via `POST /api/history/:id/model-3mf`
- Replace or delete linked files at any time
- Upload from history detail view or from inside the 3D viewer modal
- Drag-and-drop support when no model is available
- Auto-extracts `model_name` from uploaded 3MF metadata

## Printer Capabilities System
- New `server/printer-capabilities.js` ŌĆö centralised per-brand/model configuration
- Defines file access method (FTPS vs HTTP API), camera modes, features per printer type
- Model-specific overrides for camera modes (P2S, A1, X1C, Snapmaker U1)
- Eliminates scattered `if (type === 'moonraker')` checks

## Auto 3MF Caching
- Downloads and caches 3MF from Bambu printer at print start (background, non-blocking)
- Extracts `Title` and `designId` from 3MF metadata for history enrichment
- 10-second timeout ŌĆö never delays server startup or print tracking
- Cached files used for subsequent 3D preview requests

- Removed deprecated `--experimental-sqlite` flag from all files (built-in since Node 22.5+)
