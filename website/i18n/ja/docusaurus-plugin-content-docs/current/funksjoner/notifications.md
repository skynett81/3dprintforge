# アラート

Bambu Dashboardは複数のチャンネルを通じてアラートをサポートしており、自宅にいても外出先でも、プリンターの状況を常に把握できます。

アクセス先：**https://localhost:3443/#settings** → **アラート**タブ

## 利用可能なチャンネル

| チャンネル | 要件 | 画像サポート |
|---|---|---|
| Telegram | ボットトークン + チャットID | ✅ |
| Discord | Webhook URL | ✅ |
| メール | SMTPサーバー | ✅ |
| Webhook | URL + オプションのキー | ✅（base64） |
| ntfy | ntfyサーバー + トピック | ❌ |
| Pushover | APIトークン + ユーザーキー | ✅ |
| SMS（Twilio） | アカウントSID + 認証トークン | ❌ |
| ブラウザプッシュ | 設定不要 | ❌ |

## チャンネルごとの設定

### Telegram

1. [@BotFather](https://t.me/BotFather)でボットを作成 — `/newbot`を送信
2. **ボットトークン**をコピー（形式：`123456789:ABC-def...`）
3. ボットとの会話を開始して`/start`を送信
4. **チャットID**を確認：`https://api.telegram.org/bot<TOKEN>/getUpdates`にアクセス
5. Bambu Dashboardにトークンとチャットを貼り付けて、**テスト**をクリック

:::tip グループチャンネル
Telegramグループを受信者として使用できます。グループのチャットIDは`-`で始まります。
:::

### Discord

1. アラートを送りたいDiscordサーバーを開きます
2. チャンネル設定 → **インテグレーション → ウェブフック**に移動します
3. **新しいウェブフック**をクリックし、名前を付けてチャンネルを選択します
4. Webhook URLをコピーします
5. Bambu DashboardにURLを貼り付けて**テスト**をクリックします

### メール

1. SMTPサーバー、ポート（TLS用は通常587）を入力します
2. SMTPアカウントのユーザー名とパスワードを入力します
3. **送信元**アドレスと**送信先**アドレス（複数の場合はカンマ区切り）を入力します
4. 安全な送信のために**TLS/STARTTLS**を有効にします
5. **テスト**をクリックしてテストメールを送信します

:::warning Gmail
Gmailには通常のパスワードではなく**アプリパスワード**を使用してください。まずGoogleアカウントで2要素認証を有効にしてください。
:::

### ntfy

1. [ntfy.sh](https://ntfy.sh)でトピックを作成するか、独自のntfyサーバーを実行します
2. サーバーURL（例：`https://ntfy.sh`）とトピック名を入力します
3. スマートフォンにntfyアプリをインストールして同じトピックをサブスクライブします
4. **テスト**をクリックします

### Pushover

1. [pushover.net](https://pushover.net)でアカウントを作成します
2. 新しいアプリケーションを作成 — **APIトークン**をコピーします
3. PushoverダッシュボードでI**ユーザーキー**を確認します
4. 両方をBambu Dashboardに入力して**テスト**をクリックします

### Webhook（カスタム）

Bambu DashboardはJSONペイロードでHTTP POSTを送信します。

```json
{
  "event": "print_complete",
  "printer": "私のX1C",
  "printer_id": "abc123",
  "timestamp": "2026-03-22T14:30:00Z",
  "data": {
    "file": "benchy.3mf",
    "duration_minutes": 47,
    "filament_used_g": 12.4
  }
}
```

**シークレットキー**を追加して、ヘッダー`X-Bambu-Signature`のHMAC-SHA256署名でリクエストを検証します。

## イベントフィルター

チャンネルごとにアラートをトリガーするイベントを選択します。

| イベント | 説明 |
|---|---|
| プリント開始 | 新しい印刷が始まる |
| プリント完了 | 印刷完了（画像付き） |
| プリント失敗 | エラーで印刷が中断された |
| プリント一時停止 | 手動または自動の一時停止 |
| Print Guardアラート | XCamまたはセンサーがアクションをトリガー |
| フィラメント残量少 | スプールがほぼ空 |
| AMSエラー | 詰まり、湿ったフィラメントなど |
| プリンターオフライン | MQTT接続が失われた |
| キュージョブ送信 | キューからジョブがディスパッチされた |

各チャンネルごとに個別にイベントにチェックを入れてください。

## サイレント時間

夜間にアラートを受け取らないようにします。

1. アラート設定で**サイレント時間**を有効にします
2. **開始**と**終了**時刻を設定します（例：23:00 → 07:00）
3. タイマーの**タイムゾーン**を選択します
4. 重大なアラート（Print Guardエラー）は上書き可能 — **重大なものは常に送信**にチェック

## ブラウザプッシュ通知

アプリなしでブラウザに直接通知を受け取ります。

1. **設定 → アラート → ブラウザプッシュ**に移動します
2. **プッシュ通知を有効にする**をクリックします
3. ブラウザの許可ダイアログを承認します
4. ダッシュボードが最小化されていても通知が届きます（タブが開いている必要があります）

:::info PWA
バックグラウンドでのプッシュ通知（タブを開かなくても）のためにBambu DashboardをPWAとしてインストールしてください。[PWA](../system/pwa)を参照してください。
:::
