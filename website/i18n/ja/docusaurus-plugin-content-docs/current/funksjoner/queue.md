# 印刷キュー

印刷キューを使用すると、印刷を事前に計画し、プリンターが空いたときに利用可能なプリンターに自動的に送信できます。

アクセス先：**https://localhost:3443/#queue**

## キューの作成

1. ナビゲーションメニューで**印刷キュー**に移動します
2. **新しいジョブ**（+アイコン）をクリックします
3. 以下を入力します：
   - **ファイル名** — `.3mf`または`.gcode`をアップロード
   - **対象プリンター** — 特定のプリンターを選択または**自動**
   - **優先度** — 低 / 通常 / 高 / 重要
   - **計画開始時刻** — 今すぐまたは特定の日時
4. **キューに追加**をクリックします

:::tip ドラッグ＆ドロップ
ファイルエクスプローラーからキューページに直接ファイルをドラッグして素早く追加できます。
:::

## ファイルの追加

### ファイルのアップロード

1. **アップロード**をクリックするか、ファイルをアップロードフィールドにドラッグします
2. サポートされている形式：`.3mf`、`.gcode`、`.bgcode`
3. ファイルはファイルライブラリに保存され、キュージョブにリンクされます

### ファイルライブラリから

1. **ファイルライブラリ**に移動してファイルを見つけます
2. ファイルの**キューに追加**をクリックします
3. デフォルト設定でジョブが作成されます — 必要に応じて編集します

### 履歴から

1. **履歴**で過去のプリントを開きます
2. **再プリント**をクリックします
3. 前回と同じ設定でジョブが追加されます

## 優先度

キューは優先度順に処理されます。

| 優先度 | 色 | 説明 |
|---|---|---|
| 重要 | 赤 | 他のジョブに関わらず最初に空いたプリンターに送信 |
| 高 | オレンジ | 通常および低優先度のジョブより前 |
| 通常 | 青 | デフォルトの順序（FIFO） |
| 低 | 灰色 | 高優先度のジョブがない場合のみ送信 |

同じ優先度レベル内でジョブをドラッグ＆ドロップして順序を手動で変更できます。

## 自動ディスパッチ

**自動ディスパッチ**が有効になっている場合、Bambu Dashboardはすべてのプリンターを監視して次のジョブを自動的に送信します。

1. **設定 → キュー**に移動します
2. **自動ディスパッチ**をオンにします
3. **ディスパッチ戦略**を選択します：
   - **最初に空いたもの** — 空いた最初のプリンターに送信
   - **最も使用が少ないもの** — 今日のプリント数が最も少ないプリンターを優先
   - **ラウンドロビン** — すべてのプリンター間で均等にローテーション

:::warning 確認
ファイルを送信する前に各ディスパッチを手動で承認したい場合は、設定で**確認を要求**を有効にしてください。
:::

## 時差スタート

時差スタートは、すべてのプリンターが同時に開始・終了するのを避けるのに便利です。

1. **新しいジョブ**ダイアログで**詳細設定**を展開します
2. **時差スタート**を有効にします
3. **プリンター間の遅延**を設定します（例：30分）
4. システムが自動的に開始時刻を分散させます

**例：** 4つの同一ジョブが30分の遅延で午前8:00、8:30、9:00、9:30に開始されます。

## キューの状態と追跡

キューの概要には、すべてのジョブの状態が表示されます。

| 状態 | 説明 |
|---|---|
| 待機中 | ジョブはキューにあり、プリンターを待っています |
| 計画済み | 将来の開始時刻が設定されています |
| 送信中 | プリンターに転送中 |
| 印刷中 | 選択したプリンターで進行中 |
| 完了 | 完了 — 履歴にリンクされます |
| 失敗 | 送信または印刷中にエラー |
| キャンセル | 手動でキャンセルされた |

:::info アラート
キューイベントのアラートを有効にするには**設定 → アラート → キュー**から設定し、ジョブの開始、完了、失敗時に通知を受け取ってください。[アラート](./notifications)を参照してください。
:::
