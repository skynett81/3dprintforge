# バックアップと復元

3DPrintForgeはすべてのデータをローカルに保存します — 印刷履歴、フィラメント在庫、設定、ユーザーなど。定期的なバックアップにより、サーバー障害や移行時にデータを失わないようにできます。

## バックアップに含まれるものは？

| データ | 含まれる | 備考 |
|--------|---------|------|
| 印刷履歴 | はい | すべてのログと統計 |
| フィラメント在庫 | はい | スプール、重量、ブランド |
| 設定 | はい | すべてのシステム設定 |
| プリンター設定 | はい | IPアドレス、アクセスコード |
| ユーザーとロール | はい | パスワードはハッシュ化して保存 |
| 通知設定 | はい | Telegramトークンなど |
| カメラ画像 | 任意 | 大きなファイルになる可能性あり |
| タイムラプス動画 | 任意 | デフォルトでは除外 |

## 自動夜間バックアップ

デフォルトでは、毎晩03:00に自動バックアップが実行されます。

**自動バックアップの表示と設定：**
1. **システム → バックアップ**に移動します
2. **自動バックアップ**で以下が確認できます：
   - 最後に成功したバックアップと時刻
   - 次のスケジュールされたバックアップ
   - 保存されているバックアップ数（デフォルト：7日分）

**設定：**
- **時刻** — デフォルトの03:00から都合の良い時刻に変更
- **保存期間** — バックアップを保持する日数（7、14、30日）
- **保存場所** — ローカルフォルダー（デフォルト）または外部パス
- **圧縮** — デフォルトで有効（サイズを60〜80%削減）

:::info バックアップファイルはデフォルトでここに保存されます
```
/path/to/3dprintforge/data/backups/
backup-2025-03-22-030000.tar.gz
backup-2025-03-21-030000.tar.gz
...
```
:::

## 手動バックアップ

いつでもバックアップを取得できます：

1. **システム → バックアップ**に移動します
2. **今すぐバックアップ**をクリックします
3. ステータスが**完了**と表示されるまで待ちます
4. **ダウンロード**をクリックしてバックアップファイルをダウンロードします

**ターミナル経由の代替方法：**
```bash
cd /sti/til/3dprintforge
node scripts/backup.js
```

バックアップファイルはタイムスタンプ付きのファイル名で`data/backups/`に保存されます。

## バックアップからの復元

:::warning 復元は既存のデータを上書きします
すべての既存データがバックアップファイルの内容に置き換えられます。正しいファイルに復元していることを確認してください。
:::

### ダッシュボード経由

1. **システム → バックアップ**に移動します
2. **復元**をクリックします
3. リストからバックアップファイルを選択するか、ディスクからバックアップファイルをアップロードします
4. **今すぐ復元**をクリックします
5. 復元後にダッシュボードが自動的に再起動します

### ターミナル経由

```bash
cd /sti/til/3dprintforge
node scripts/restore.js data/backups/backup-2025-03-22-030000.tar.gz
```

復元後、ダッシュボードを再起動します：
```bash
sudo systemctl restart 3dprintforge
# または
npm start
```

## 設定のエクスポートとインポート

設定だけを保存したい場合（すべての履歴ではなく）？

**エクスポート：**
1. **システム → 設定 → エクスポート**に移動します
2. 含めるものを選択します：
   - プリンター設定
   - 通知設定
   - ユーザーアカウント
   - フィラメントブランドとプロファイル
3. **エクスポート**をクリック — `.json`ファイルをダウンロードします

**インポート：**
1. **システム → 設定 → インポート**に移動します
2. `.json`ファイルをアップロードします
3. インポートする部分を選択します
4. **インポート**をクリックします

:::tip 新規インストールに便利
エクスポートした設定は新しいサーバーへの移行時に便利です。新規インストール後にインポートして最初から設定し直す必要をなくします。
:::

## 新しいサーバーへの移行

すべてのデータを新しいマシンに3DPrintForgeと共に移行する方法：

### ステップ1 — 古いサーバーでバックアップを取る

1. **システム → バックアップ → 今すぐバックアップ**に移動します
2. バックアップファイルをダウンロードします
3. ファイルを新しいサーバーにコピーします（USB、scp、ネットワーク共有）

### ステップ2 — 新しいサーバーにインストール

```bash
git clone https://github.com/skynett81/3dprintforge.git
cd 3dprintforge
./install.sh
```

インストールガイドに従ってください。何も設定する必要はありません — ダッシュボードを起動して実行するだけです。

### ステップ3 — バックアップを復元する

新しいサーバーでダッシュボードが実行されたら：

1. **システム → バックアップ → 復元**に移動します
2. 古いサーバーからバックアップファイルをアップロードします
3. **今すぐ復元**をクリックします

これで履歴、フィラメント在庫、設定、ユーザーがすべて揃います。

### ステップ4 — 接続を確認する

1. **設定 → プリンター**に移動します
2. 各プリンターへの接続をテストします
3. IPアドレスがまだ正しいか確認します（新しいサーバーは異なるIPを持つ可能性があります）

## バックアップのベストプラクティス

- **復元をテストする** — バックアップを取ってテストマシンで復元してください（少なくとも一度）。テストされていないバックアップはバックアップではありません。
- **外部に保存する** — 定期的にバックアップファイルを外部ディスクやクラウドストレージ（Nextcloud、Google Driveなど）にコピーします
- **アラートを設定する** — **設定 → 通知 → イベント**で「バックアップ失敗」の通知を有効にして、何かがうまくいかない場合にすぐにわかるようにします
