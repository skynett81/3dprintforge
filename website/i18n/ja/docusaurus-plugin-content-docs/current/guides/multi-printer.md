# 複数のプリンター

プリンターが複数台ありますか？3DPrintForgeはフリート管理のために作られています — 一か所からすべてのプリンターを監視、制御、調整できます。

## 新しいプリンターを追加する

1. **設定 → プリンター**に移動します
2. **+ プリンターを追加**をクリックします
3. 入力します：

| フィールド | 例 | 説明 |
|-----------|-----|------|
| シリアル番号 (SN) | 01P... | Bambu Handyまたはプリンターの画面で確認 |
| IPアドレス | 192.168.1.101 | LANモード用（推奨） |
| アクセスコード | 12345678 | プリンター画面の8桁コード |
| 名前 | "Bambu #2 - P1S" | ダッシュボードに表示される |
| モデル | P1P, P1S, X1C, A1 | 正しいアイコンと機能のために正しいモデルを選択 |

4. **接続テスト**をクリック — 緑のステータスが表示されるはずです
5. **保存**をクリックします

:::tip プリンターにわかりやすい名前をつける
「Bambu 1」と「Bambu 2」は混乱しやすいです。「X1C - 生産」や「P1S - プロトタイプ」のような名前を使うと管理しやすくなります。
:::

## フリート概要

すべてのプリンターを追加した後、**フリート**パネルにまとめて表示されます。ここで確認できます：

```
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│ X1C - 生産      │  │ P1S - プロトタイプ│  │ A1 - 趣味部屋   │
│ ████████░░ 82%  │  │ アイドル         │  │ ████░░░░░░ 38%  │
│ 1時間24分残り   │  │ 印刷準備完了     │  │ 3時間12分残り   │
│ 温度: 220/60°C  │  │ AMS: 4スプール  │  │ 温度: 235/80°C  │
└─────────────────┘  └─────────────────┘  └─────────────────┘
```

以下ができます：
- プリンターをクリックして詳細ビューを表示
- すべての温度、AMSステータス、アクティブなエラーを一度に確認
- ステータスでフィルタリング（アクティブな印刷、アイドル、エラー）

## 印刷キュー — 作業の分散

印刷キューで一か所からすべてのプリンターへの印刷を計画できます。

**仕組み：**
1. **キュー**に移動します
2. **+ ジョブを追加**をクリックします
3. ファイルと設定を選択します
4. プリンターを選択するか、**自動割り当て**を選択します

### 自動割り当て
自動割り当てでは、ダッシュボードが以下に基づいてプリンターを選択します：
- 空き容量
- AMSで利用可能なフィラメント
- 計画されたメンテナンスウィンドウ

**設定 → キュー → 自動割り当て**で有効にします。

### 優先度
キューのジョブをドラッグアンドドロップして順序を変更します。**高優先度**ジョブは通常のジョブより前に移動します。

## スタッガードスタート — 電力ピークを避ける

多くのプリンターを同時に起動すると、ウォームアップフェーズで強い電力ピークが発生する可能性があります。スタッガードスタートは起動を分散します：

**有効化する方法：**
1. **設定 → フリート → スタッガードスタート**に移動します
2. **分散起動**を有効にします
3. プリンター間の遅延を設定します（推奨：2〜5分）

**3台のプリンターと3分の遅延の例：**
```
08:00 — プリンター1がウォームアップ開始
08:03 — プリンター2がウォームアップ開始
08:06 — プリンター3がウォームアップ開始
```

:::tip ブレーカーサイズに関連
X1Cはウォームアップ中に約1000Wを消費します。3台同時 = 3000W、16Aブレーカーが作動する可能性があります。スタッガードスタートで問題を解消できます。
:::

## プリンターグループ

プリンターグループでプリンターを論理的に整理し、グループ全体にコマンドを送ることができます：

**グループの作成：**
1. **設定 → プリンターグループ**に移動します
2. **+ 新しいグループ**をクリックします
3. グループに名前をつけます（例：「生産フロア」、「趣味部屋」）
4. グループにプリンターを追加します

**グループ機能：**
- グループの集計統計を確認
- グループ全体に一斉に一時停止コマンドを送信
- グループのメンテナンスウィンドウを設定

## すべてのプリンターを監視する

### マルチカメラビュー
**フリート → カメラビュー**に移動して、すべてのカメラフィードを並べて確認します：

```
┌──────────────┐  ┌──────────────┐
│  X1C フィード │  │  P1S フィード │
│  [ライブ]    │  │  [アイドル]  │
└──────────────┘  └──────────────┘
┌──────────────┐  ┌──────────────┐
│  A1 フィード  │  │  + 追加      │
│  [ライブ]    │  │              │
└──────────────┘  └──────────────┘
```

### プリンター別通知
異なるプリンターに異なる通知ルールを設定できます：
- 生産用プリンター：夜間も含めて常に通知
- 趣味用プリンター：日中のみ通知

設定については[通知](./notification-setup)を参照してください。

## フリート運用のヒント

- **フィラメントスロットを標準化する**：すべてのプリンターでスロット1にPLAホワイト、スロット2にPLAブラックを保持 — ジョブ配分が簡単になります
- **AMSレベルを毎日確認する**：朝のルーティンについては[日常的な使い方](./daily-use)を参照
- **ローテーションでメンテナンス**：すべてのプリンターを同時にメンテナンスしない — 常に少なくとも1台は稼働させておく
- **ファイルをわかりやすく命名する**：`logo_x1c_pla_0.2mm.3mf`のようなファイル名で適切なプリンターを選びやすくなります
