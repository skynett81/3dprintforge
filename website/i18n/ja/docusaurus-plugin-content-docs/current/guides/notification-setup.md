# 通知の設定

Bambu Dashboardは印刷完了から重大なエラーまで、あらゆることをTelegram、Discord、メール、またはブラウザプッシュ通知でお知らせできます。

## 通知チャンネルの概要

| チャンネル | 最適な用途 | 必要なもの |
|-----------|-----------|-----------|
| Telegram | 素早く、どこでも | Telegramアカウント + ボットトークン |
| Discord | チーム/コミュニティ | Discordサーバー + Webhook URL |
| メール (SMTP) | 公式通知 | SMTPサーバー |
| ブラウザプッシュ | デスクトップ通知 | プッシュ対応ブラウザ |

---

## Telegramボット

### ステップ1 — ボットを作成する

1. Telegramを開いて**@BotFather**を検索します
2. `/newbot`を送信します
3. ボットに名前をつけます（例：「Bambu 通知」）
4. ボットにユーザー名をつけます（例：`bambu_notification_bot`）— `bot`で終わる必要があります
5. BotFatherが**APIトークン**を送信します：`123456789:ABCdefGHIjklMNOpqrsTUVwxyz`
6. このトークンをコピーして保存します

### ステップ2 — Chat IDを見つける

1. ボットとの会話を始めます（ユーザー名を検索して**スタート**をクリック）
2. ボットにメッセージを送ります（例：「こんにちは」）
3. ブラウザで`https://api.telegram.org/bot<あなたのトークン>/getUpdates`に移動します
4. `"chat":{"id": 123456789}`を見つけます — それがあなたのChat IDです

### ステップ3 — ダッシュボードに接続する

1. **設定 → 通知 → Telegram**に移動します
2. **ボットトークン**を貼り付けます
3. **Chat ID**を貼り付けます
4. **通知テスト**をクリック — Telegramにテストメッセージが届くはずです
5. **保存**をクリックします

:::tip グループ通知
グループ全体に通知したいですか？ボットをTelegramグループに追加し、グループのChat ID（負の数、例：`-100123456789`）を見つけて代わりに使用してください。
:::

## Discord Webhook

### ステップ1 — DiscordでWebhookを作成する

1. Discordサーバーに移動します
2. 通知を受け取りたいチャンネルを右クリック → **チャンネルの編集**
3. **連携サービス → ウェブフック**に移動します
4. **ウェブフックを作成**をクリックします
5. 名前をつけます（例：「Bambu Dashboard」）
6. アバターを選びます（任意）
7. **ウェブフックのURLをコピー**をクリックします

URLはこのような形式です：
```
https://discord.com/api/webhooks/123456789/abcdefghijk...
```

### ステップ2 — ダッシュボードに入力する

1. **設定 → 通知 → Discord**に移動します
2. **Webhook URL**を貼り付けます
3. **通知テスト**をクリック — Discordチャンネルにテストメッセージが届くはずです
4. **保存**をクリックします

## メール (SMTP)

### 必要な情報

メールプロバイダーのSMTP設定が必要です：

| プロバイダー | SMTPサーバー | ポート | 暗号化 |
|------------|------------|-------|-------|
| Gmail | smtp.gmail.com | 587 | TLS |
| Outlook/Hotmail | smtp-mail.outlook.com | 587 | TLS |
| Yahoo | smtp.mail.yahoo.com | 587 | TLS |
| 独自ドメイン | smtp.yourdomian.jp | 587 | TLS |

:::warning GmailはAppパスワードが必要です
Gmailは通常のパスワードでのログインをブロックします。Googleアカウント → セキュリティ → 2段階認証 → アプリパスワードで**アプリパスワード**を作成する必要があります。
:::

### ダッシュボードでの設定

1. **設定 → 通知 → メール**に移動します
2. 入力します：
   - **SMTPサーバー**：例 `smtp.gmail.com`
   - **ポート**：`587`
   - **ユーザー名**：メールアドレス
   - **パスワード**：アプリパスワードまたは通常のパスワード
   - **送信元アドレス**：通知が送られるメール
   - **宛先アドレス**：通知を受け取るメール
3. **テストメール**をクリックします
4. **保存**をクリックします

## ブラウザプッシュ通知

プッシュ通知はブラウザタブがバックグラウンドにある場合でもデスクトップにシステム通知として表示されます。

**有効化：**
1. **設定 → 通知 → プッシュ通知**に移動します
2. **プッシュ通知を有効にする**をクリックします
3. ブラウザが許可を求めます — **許可**をクリックします
4. **通知テスト**をクリックします

:::info 有効にしたブラウザでのみ
プッシュ通知は特定のブラウザとデバイスに紐付けられています。通知を受け取りたい各デバイスで有効にしてください。
:::

## 通知するイベントの選択

通知チャンネルを設定したら、どのイベントが通知をトリガーするかを正確に選択できます：

**設定 → 通知 → イベントの下：**

| イベント | 推奨 |
|---------|------|
| 印刷完了 | はい |
| 印刷失敗 / キャンセル | はい |
| Print Guard：スパゲッティ検出 | はい |
| HMSエラー（重大） | はい |
| HMS警告 | 任意 |
| フィラメント残量低下 | はい |
| AMSエラー | はい |
| プリンター切断 | 任意 |
| メンテナンスリマインダー | 任意 |
| 夜間バックアップ完了 | いいえ（煩わしい） |

## サイレントタイム（夜間に通知しない）

午前3時に印刷完了で起こされないようにします：

1. **設定 → 通知 → サイレントタイム**に移動します
2. **サイレントタイム**を有効にします
3. 開始時刻と終了時刻を設定します（例：**22:00〜07:00**）
4. サイレント期間中も通知するイベントを選択します：
   - **重大なHMSエラー** — 有効のままにすることを推奨
   - **Print Guard** — 有効のままにすることを推奨
   - **印刷完了** — 夜間はオフにできます

:::tip 夜間印刷を妨げずに
サイレントタイムを有効にして夜間に印刷します。Print Guardが監視します — そして朝にサマリーが届きます。
:::
