# 電力測定

エネルギー測定機能付きのスマートプラグをプリンターに接続して、プリントごとの実際の電力消費量を記録します — 推定値だけでなく。

アクセス先：**https://localhost:3443/#settings** → **統合 → 電力測定**

## サポートされているデバイス

| デバイス | プロトコル | 推奨度 |
|---|---|---|
| **Shelly Plug S / Plus Plug S** | HTTP REST / MQTT | 推奨 — 簡単なセットアップ |
| **Shelly 1PM / 2PM** | HTTP REST / MQTT | 固定インストール向け |
| **Shelly Gen2 / Gen3** | HTTP REST / MQTT | 拡張APIを持つ新しいモデル |
| **Tasmotaデバイス** | MQTT | カスタムビルドセットアップに柔軟 |

:::tip 推奨デバイス
ファームウェア1.0以降のShelly Plug S Plusはテスト済みで推奨されています。クラウド依存なしでWi-Fi、MQTT、HTTP RESTをサポートします。
:::

## Shellyによるセットアップ

### 前提条件

- Shellyプラグが3DPrintForgeと同じネットワークに接続されている
- Shellyが静的IPまたはDHCP予約で設定されている

### 設定

1. **設定 → 電力測定**に移動します
2. **電力メーターを追加**をクリックします
3. **タイプ**を選択：Shelly
4. 以下を入力します：
   - **IPアドレス**：例 `192.168.1.150`
   - **チャンネル**：0（シングルコンセントプラグの場合）
   - **認証**：設定されている場合はユーザー名とパスワード
5. **接続テスト**をクリックします
6. プラグを**プリンター**にリンク：ドロップダウンリストから選択
7. **保存**をクリックします

### ポーリング間隔

デフォルトのポーリング間隔は10秒です。より正確な測定には5秒に短縮、ネットワーク負荷軽減には30秒に延長します。

## Tasmotaによるセットアップ

1. TasmotaデバイスをMQTTで設定します（Tasmotaドキュメントを参照）
2. 3DPrintForgeで：**タイプ**：Tasmotaを選択
3. デバイスのMQTTトピックを入力：例 `tasmota/power-plug-1`
4. プリンターにリンクして**保存**をクリックします

3DPrintForgeが電力測定のために`{topic}/SENSOR`を自動的にサブスクライブします。

## 何が測定されるか

電力測定が有効な場合、プリントごとに以下が記録されます。

| 指標 | 説明 |
|---|---|
| **瞬時電力** | プリント中のワット数（リアルタイム） |
| **総エネルギー消費量** | プリント全体のkWh |
| **平均電力** | kWh / プリント時間 |
| **エネルギーコスト** | kWh × 電力料金（Tibber/Nordpoolから） |

データはプリント履歴に保存され、分析に利用できます。

## ライブ表示

瞬時電力消費量は以下に表示されます。

- **ダッシュボード** — 追加ウィジェットとして（ウィジェット設定で有効化）
- **フリート概要** — プリンターカードの小さなインジケーター

## 推定値との比較

プリント後に比較が表示されます。

| | 推定 | 実測 |
|---|---|---|
| エネルギー消費量 | 1.17 kWh | 1.09 kWh |
| 電力コスト | 2.16円 | 2.02円 |
| 偏差 | — | -6.8% |

一貫した偏差は[コスト計算機](../analytics/costestimator)の推定値を調整するために使用できます。

## プリンターを自動的にオフにする

Shelly/Tasmotaはプリント完了後にプリンターを自動的にオフにできます。

1. **電力測定 → [プリンター] → 自動オフ**に移動します
2. **プリント完了後X分でオフにする**を有効にします
3. 時間遅延を設定します（例：10分）

:::danger 冷却
電源を切る前にプリンターを少なくとも5〜10分間冷却させてください。ホットエンドの熱クリープを防ぐため、ノズルが50°C未満まで冷却される必要があります。
:::
