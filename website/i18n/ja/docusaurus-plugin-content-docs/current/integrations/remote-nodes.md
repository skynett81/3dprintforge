# リモートサーバー（Remote Nodes）

リモートサーバー機能を使用すると、複数の3DPrintForgeインスタンスを接続して、同じネットワーク上でも異なる場所でも、1つの中央インターフェースからすべてのプリンターを表示・制御できます。

アクセス先：**https://localhost:3443/#settings** → **統合 → リモートサーバー**

## ユースケース

- **自宅 + オフィス** — 同じダッシュボードから両方の場所のプリンターを表示
- **メイカースペース** — スペース内のすべてのインスタンスの中央ダッシュボード
- **ゲストインスタンス** — フルアクセスなしで顧客に限定的な閲覧権限を付与

## アーキテクチャ

```
プライマリインスタンス（あなたのPC）
  ├── プリンターA（ローカルMQTT）
  ├── プリンターB（ローカルMQTT）
  └── リモートサーバー：セカンダリインスタンス
        ├── プリンターC（リモートロケーションのMQTT）
        └── プリンターD（リモートロケーションのMQTT）
```

プライマリインスタンスはREST APIを通じてリモートサーバーをポーリングし、ローカルにデータを集約します。

## リモートサーバーを追加する

### ステップ1：リモートインスタンスでAPIキーを生成する

1. リモートインスタンスにログインします（例：`https://192.168.2.50:3443`）
2. **設定 → APIキー**に移動します
3. **新しいキー**をクリックして「プライマリノード」という名前を付けます
4. 権限を設定：**読み取り**（最低限）または**読み取り + 書き込み**（リモート制御の場合）
5. キーをコピーします

### ステップ2：プライマリインスタンスから接続する

1. **設定 → リモートサーバー**に移動します
2. **リモートサーバーを追加**をクリックします
3. 以下を入力します：
   - **名前**：例「オフィス」または「ガレージ」
   - **URL**：`https://192.168.2.50:3443`または外部URL
   - **APIキー**：ステップ1のキー
4. **接続テスト**をクリックします
5. **保存**をクリックします

:::warning 自己署名証明書
リモートインスタンスが自己署名証明書を使用している場合、**TLSエラーを無視**を有効にします — ただし、内部ネットワーク接続にのみ行ってください。
:::

## 集約表示

接続後、リモートプリンターは以下に表示されます。

- **フリート概要** — リモートサーバー名とクラウドアイコン付き
- **統計** — すべてのインスタンスにわたって集約
- **フィラメント在庫** — 統合された概要

## リモート制御

**読み取り + 書き込み**権限があると、リモートプリンターを直接制御できます。

- 一時停止 / 再開 / 停止
- プリントキューへの追加（ジョブがリモートインスタンスに送信される）
- カメラストリームの表示（リモートインスタンス経由でプロキシ）

:::info レイテンシ
リモートサーバー経由のカメラストリームは、ネットワーク速度と距離によって顕著な遅延が発生する場合があります。
:::

## アクセス制御

リモートサーバーが共有するデータを制限します。

1. リモートインスタンスで：**設定 → APIキー → [キー名]**に移動します
2. アクセスを制限します：
   - 特定のプリンターのみ
   - カメラストリームなし
   - 書き込み保護（読み取りのみ）

## ヘルスと監視

各リモートサーバーのステータスが**設定 → リモートサーバー**に表示されます。

- **接続済み** — 最後のポーリングが成功
- **切断済み** — リモートサーバーに到達できない
- **認証エラー** — APIキーが無効または期限切れ
- **最終同期** — 最後に成功したデータ同期のタイムスタンプ
