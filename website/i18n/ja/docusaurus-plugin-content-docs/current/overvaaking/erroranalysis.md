# エラーパターン分析

エラーパターン分析は、プリントとエラーの履歴データを使用してパターン、原因、相関関係を特定し、具体的な改善提案を提供します。

アクセス先：**https://localhost:3443/#error-analysis**

## 分析される内容

システムは以下のデータポイントを分析します。

- HMSエラーコードと発生時刻
- エラー発生時のフィラメントタイプとメーカー
- エラー発生時の温度（ノズル、ベッド、チャンバー）
- プリント速度とプロファイル
- 時刻と曜日
- 最後のメンテナンスからの経過時間
- プリンターモデルとファームウェアバージョン

## 相関分析

システムはエラーと要因の統計的相関を探します。

**検出される相関の例：**
- 「AMS詰まりエラーの78%がメーカーXのフィラメントで発生する」
- 「ノズル詰まりは連続印刷6時間以上後に3倍頻繁に発生する」
- 「チャンバー温度が18°C未満の場合に接着エラーが増加する」
- 「湿度60%以上でストリングエラーが相関する（湿度計が接続されている場合）」

統計的有意性のある相関（p < 0.05）が上位に表示されます。

:::info データ要件
分析は履歴に最低50プリントがある場合に最も正確です。プリント数が少ない場合は低信頼度の推定値が表示されます。
:::

## 改善提案

分析に基づいて具体的な提案が生成されます。

| 提案タイプ | 例 |
|---|---|
| フィラメント | 「PA-CFに別のメーカーに切り替えてください — エラーの3件中4件がメーカーXを使用していました」 |
| 温度 | 「PETGのベッド温度を5°C上げてください — 接着エラーが推定60%減少します」 |
| 速度 | 「4時間後に速度を80%に下げてください — ノズル詰まりが推定45%減少します」 |
| メンテナンス | 「エクストルーダーギアをクリーニングしてください — 消耗度が押し出しエラーの40%と相関しています」 |
| キャリブレーション | 「ベッドレベリングを実行してください — 先週の接着エラー15件中12件がキャリブレーションの誤りと相関しています」 |

各提案には以下が表示されます。
- 推定効果（エラーの%削減）
- 信頼度（低 / 中 / 高）
- ステップごとの実装方法
- 関連ドキュメントへのリンク

## 健康スコアへの影響

分析は健康スコアにリンクされています（[診断](./diagnostics)を参照）。

- スコアを最も下げている要因を表示
- 各提案を実装した場合のスコア改善を推定
- スコア改善の可能性に応じて提案を優先順位付け

## タイムライン表示

**エラー分析 → タイムライン**で時系列の概要を確認します。

1. プリンターと期間を選択します
2. エラーがタイムラインに種類ごとに色分けされたポイントとして表示されます
3. 水平線でメンテナンスタスクがマークされます
4. エラーのクラスター（短時間に多くのエラー）が赤でハイライトされます

クラスターをクリックしてその特定の期間の分析を開きます。

## レポート

エラー分析のPDFレポートを生成します。

1. **レポートを生成**をクリックします
2. 期間を選択します（例：過去90日間）
3. 内容を選択：相関関係、提案、タイムライン、健康スコア
4. PDFをダウンロードするかメールで送信します

レポートは、プリンターがプロジェクトにリンクされている場合はプロジェクト下に保存されます。

:::tip 週次レビュー
**設定 → レポート**で自動的な週次メールレポートを設定し、ダッシュボードを手動で訪問せずに最新情報を把握してください。[レポート](../system/reports)を参照してください。
:::
