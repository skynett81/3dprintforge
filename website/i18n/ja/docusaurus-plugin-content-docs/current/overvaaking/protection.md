# Print Guard

Print GuardはBambu Dashboardのリアルタイム監視システムです。カメラ、センサー、プリンターデータを継続的に監視し、問題が発生したときに設定されたアクションを実行します。

アクセス先：**https://localhost:3443/#protection**

## XCamイベント検出

Bambu LabプリンターはAIカメラが問題を検出するとMQTT経由でXCamイベントを送信します。

| イベント | コード | 深刻度 |
|---|---|---|
| スパゲッティ検出 | `xcam_spaghetti` | 重大 |
| プレートの剥離 | `xcam_detach` | 高 |
| 第1レイヤーの不具合 | `xcam_first_layer` | 高 |
| ストリング | `xcam_stringing` | 中 |
| 押し出しエラー | `xcam_extrusion` | 高 |

各イベントタイプに対して1つまたは複数のアクションを設定できます。

- **アラート** — アクティブなアラートチャンネルで通知を送信
- **一時停止** — 手動確認のためにプリントを一時停止
- **停止** — プリントを直ちに中断
- **なし** — イベントを無視する（ただしログには記録）

:::danger デフォルト動作
デフォルトでは、XCamイベントは**アラート**と**一時停止**に設定されています。AI検出を完全に信頼する場合は**停止**に変更してください。
:::

## センサー監視

Print Guardはセンサーデータを継続的に監視し、異常時にアラームを発します。

### 温度偏差

1. **Print Guard → 温度**に移動します
2. **目標温度からの最大偏差**を設定します（推奨：ノズル±5°C、ベッド±3°C）
3. 偏差時の**アクション**を選択：アラート / 一時停止 / 停止
4. アクション実行前の**遅延**（秒）を設定 — 温度が安定するまでの時間を確保

### フィラメント残量少

システムはスプールの残りフィラメントを計算します。

1. **Print Guard → フィラメント**に移動します
2. **最小限度**をグラムで設定します（例：50 g）
3. アクションを選択：手動でスプールを交換するために**一時停止してアラート**（推奨）

### プリント停止検出

予期しないプリントの停止（MQTTタイムアウト、フィラメント切れなど）を検出します。

1. **停止検出**を有効にします
2. **タイムアウト**を設定します（推奨：データなしで120秒 = 停止）
3. アクション：常にアラート — プリンターはすでに停止している可能性があります

## 設定

### Print Guardの有効化

1. **設定 → Print Guard**に移動します
2. **Print Guardを有効にする**をオンにします
3. 監視するプリンターを選択します
4. **保存**をクリックします

### プリンターごとのルール

プリンターごとに異なるルールを設定できます。

1. Print Guard概要のプリンターをクリックします
2. **グローバルルールを継承**をオフにします
3. このプリンター専用のルールを設定します

## ログとイベント履歴

すべてのPrint Guardイベントがログに記録されます。

- **Print Guard → ログ**に移動します
- プリンター、イベントタイプ、日付、深刻度でフィルタリングします
- イベントをクリックして詳細情報と実行されたアクションを確認します
- ログをCSVにエクスポートします

:::tip 誤検知
Print Guardが不要な一時停止をトリガーする場合は、**Print Guard → 設定 → 感度**で感度を調整してください。「低」から始めて徐々に上げてください。
:::
