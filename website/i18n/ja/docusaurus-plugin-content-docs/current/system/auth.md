# 認証

3DPrintForgeはロールベースのアクセス制御、APIキー、オプションのTOTPによる2要素認証（2FA）を備えたマルチユーザーをサポートします。

アクセス先：**https://localhost:3443/#settings** → **ユーザーとアクセス**

## ユーザー

### ユーザーの作成

1. **設定 → ユーザー**に移動します
2. **新しいユーザー**をクリックします
3. 以下を入力します：
   - **ユーザー名** — ログインに使用
   - **メールアドレス**
   - **パスワード** — 最低12文字を推奨
   - **ロール** — 以下のロールを参照
4. **作成**をクリックします

新しいユーザーは**https://localhost:3443/login**からログインできます。

### パスワードの変更

1. **プロファイル**（右上隅 → ユーザー名をクリック）に移動します
2. **パスワードを変更**をクリックします
3. 現在のパスワードと新しいパスワードを入力します
4. **保存**をクリックします

管理者は**設定 → ユーザー → [ユーザー] → パスワードをリセット**から他のユーザーのパスワードをリセットできます。

## ロール

| ロール | 説明 |
|---|---|
| **管理者** | フルアクセス — すべての設定、ユーザー、機能 |
| **オペレーター** | プリンターの制御、すべての閲覧、システム設定の変更不可 |
| **ゲスト** | 読み取りのみ — ダッシュボード、履歴、統計の閲覧 |
| **APIユーザー** | APIアクセスのみ — Webインターフェースなし |

### カスタムロール

1. **設定 → ロール**に移動します
2. **新しいロール**をクリックします
3. 権限を個別に選択します：
   - ダッシュボード / 履歴 / 統計の閲覧
   - プリンターの制御（一時停止/停止/開始）
   - フィラメント在庫の管理
   - キューの管理
   - カメラストリームの閲覧
   - 設定の変更
   - ユーザーの管理
4. **保存**をクリックします

## APIキー

APIキーはログインなしのプログラマティックアクセスを提供します。

### APIキーの作成

1. **設定 → APIキー**に移動します
2. **新しいキー**をクリックします
3. 以下を入力します：
   - **名前** — わかりやすい名前（例：「Home Assistant」、「Pythonスクリプト」）
   - **有効期限** — オプション、セキュリティのために設定
   - **権限** — ロールまたは特定の権限を選択
4. **生成**をクリックします
5. **キーを今すぐコピー** — 一度しか表示されません

### APIキーの使用

すべてのAPI呼び出しのHTTPヘッダーに追加します：
```
Authorization: Bearer あなたのAPIキー
```

テストには[APIプレイグラウンド](../tools/playground)を参照してください。

:::danger セキュアな保管
APIキーは関連付けられたユーザーと同じアクセス権を持ちます。安全に保管し、定期的にローテーションしてください。
:::

## TOTP 2FA

認証アプリ（Google Authenticator、Authy、Bitwardenなど）で2要素認証を有効にします。

### 2FAを有効にする

1. **プロファイル → セキュリティ → 2要素認証**に移動します
2. **2FAを有効にする**をクリックします
3. 認証アプリでQRコードをスキャンします
4. 生成された6桁のコードを入力して確認します
5. **リカバリーコード**（10のワンタイムコード）を安全な場所に保存します
6. **有効にする**をクリックします

### 2FAでログインする

1. ユーザー名とパスワードを通常通り入力します
2. アプリから6桁のTOTPコードを入力します
3. **ログイン**をクリックします

### すべてのユーザーに2FAを強制する

管理者はすべてのユーザーに2FAを要求できます。

1. **設定 → セキュリティ → 2FAを強制**に移動します
2. 設定を有効にします
3. 2FAなしのユーザーは次回ログイン時にセットアップを強制されます

## セッション管理

- デフォルトのセッション期間：24時間
- **設定 → セキュリティ → セッション期間**で調整
- ユーザーごとにアクティブなセッションを表示して個別のセッションを終了
