# バックアップ

3DPrintForgeはすべての設定、履歴、データをバックアップして、システム障害、サーバー移行、アップデートの問題が発生した際に簡単に復元できます。

アクセス先：**https://localhost:3443/#settings** → **システム → バックアップ**

## バックアップに含まれるもの

| データタイプ | 含まれる | 備考 |
|---|---|---|
| プリンターの設定と構成 | ✅ | |
| プリント履歴 | ✅ | |
| フィラメント在庫 | ✅ | |
| ユーザーとロール | ✅ | パスワードはハッシュ化して保存 |
| 設定 | ✅ | 通知設定を含む |
| メンテナンスログ | ✅ | |
| プロジェクトと請求書 | ✅ | |
| ファイルライブラリ（メタデータ） | ✅ | |
| ファイルライブラリ（ファイル） | オプション | 大きくなる可能性があります |
| タイムラプス動画 | オプション | 非常に大きくなる可能性があります |
| ギャラリー画像 | オプション | |

## 手動バックアップを作成する

1. **設定 → バックアップ**に移動します
2. 含めるものを選択します（上の表を参照）
3. **今すぐバックアップを作成**をクリックします
4. バックアップ作成中に進捗インジケーターが表示されます
5. バックアップ完了後に**ダウンロード**をクリックします

バックアップはファイル名にタイムスタンプ付きの`.zip`ファイルとして保存されます：
```
3dprintforge-backup-2026-03-22T14-30-00.zip
```

## バックアップのダウンロード

バックアップファイルはサーバーのバックアップフォルダーに保存されます（設定可能）。直接ダウンロードもできます：

1. **バックアップ → 既存のバックアップ**に移動します
2. リストのバックアップを検索します（日付順に並び替え）
3. **ダウンロード**（ダウンロードアイコン）をクリックします

:::info 保存フォルダー
デフォルトの保存フォルダー：`./data/backups/`。**設定 → バックアップ → 保存フォルダー**で変更します。
:::

## 自動バックアップのスケジュール設定

1. **バックアップ → スケジュール**の下で**自動バックアップ**を有効にします
2. 間隔を選択します：
   - **毎日** — 03:00に実行（設定可能）
   - **毎週** — 特定の曜日と時刻
   - **毎月** — 月の最初の日
3. **保持するバックアップ数**を選択します（例：7 — 古いものは自動削除）
4. **保存**をクリックします

:::tip 外部ストレージ
重要なデータに対して：バックアップの保存フォルダーとして外部ディスクまたはネットワークドライブをマウントします。これにより、システムディスクが故障してもバックアップが残ります。
:::

## バックアップから復元する

:::warning 復元すると既存のデータが上書きされます
復元はすべての既存データをバックアップファイルの内容に置き換えます。まず現在のデータの新しいバックアップがあることを確認してください。
:::

### サーバー上の既存バックアップから

1. **バックアップ → 既存のバックアップ**に移動します
2. リストのバックアップを検索します
3. **復元**をクリックします
4. ダイアログで確認します
5. 復元後にシステムが自動的に再起動します

### ダウンロードしたバックアップファイルから

1. **バックアップをアップロード**をクリックします
2. コンピューターから`.zip`ファイルを選択します
3. ファイルが検証されます — 含まれているものが表示されます
4. **ファイルから復元**をクリックします
5. ダイアログで確認します

## バックアップの検証

3DPrintForgeは復元前にすべてのバックアップファイルを検証します：

- ZIPフォーマットが有効であることを確認
- データベーススキーマが現在のバージョンと互換性があることを検証
- バックアップが古いバージョンからの場合は警告を表示（移行が自動的に実行されます）
