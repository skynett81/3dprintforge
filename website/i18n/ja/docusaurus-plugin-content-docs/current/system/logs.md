# サーバーログ

サーバーログは3DPrintForgeの内部で何が起きているかを把握するために役立ちます — トラブルシューティング、監視、診断に便利です。

アクセス先：**https://localhost:3443/#logs**

## リアルタイム表示

ログストリームはWebSocket経由でリアルタイムに更新されます。

1. **システム → サーバーログ**に移動します
2. 新しいログラインが自動的に下部に表示されます
3. **最下部にロック**をクリックして常に最新ログにスクロール
4. **フリーズ**をクリックして自動スクロールを停止し、既存のラインを読む

デフォルト表示では最新500行のログが表示されます。

## ログレベル

各ログラインにはレベルがあります。

| レベル | 色 | 説明 |
|---|---|---|
| **ERROR** | 赤 | 機能に影響するエラー |
| **WARN** | オレンジ | 警告 — 何かが悪化する可能性がある |
| **INFO** | 青 | 通常の運用情報 |
| **DEBUG** | グレー | 詳細な開発者情報 |

:::info ログレベルの設定
**設定 → システム → ログレベル**でログレベルを変更します。通常の運用には**INFO**を使用します。**DEBUG**はトラブルシューティング時のみ使用してください — 大量のデータが生成されます。
:::

## フィルタリング

ログビュー上部のフィルターツールバーを使用します。

1. **ログレベル** — ERROR / WARN / INFO / DEBUG のみ表示、または組み合わせ
2. **モジュール** — システムモジュールでフィルタリング：
   - `mqtt` — プリンターとのMQTT通信
   - `api` — APIリクエスト
   - `db` — データベース操作
   - `auth` — 認証イベント
   - `queue` — プリントキューイベント
   - `guard` — Print Guardイベント
   - `backup` — バックアップ操作
3. **フリーテキスト** — ログテキストを検索（正規表現をサポート）
4. **時刻** — 日付範囲でフィルタリング

精確なトラブルシューティングのためにフィルターを組み合わせます。

## 一般的なエラーシナリオ

### MQTT接続の問題

`mqtt`モジュールのログラインを確認します：

```
ERROR [mqtt] プリンター XXXXへの接続に失敗：Connection refused
```

**解決策：** プリンターがオンになっていること、アクセスキーが正しいこと、ネットワークが機能していることを確認します。

### データベースエラー

```
ERROR [db] マイグレーション v95 が失敗：SQLITE_CONSTRAINT
```

**解決策：** バックアップを取り、**設定 → システム → データベースを修復**でデータベース修復を実行します。

### 認証エラー

```
WARN [auth] IPアドレス 192.168.1.x からユーザー admin のログインに失敗
```

多くのログイン失敗はブルートフォース攻撃を示す可能性があります。IPホワイトリストを有効にするかどうかを確認してください。

## ログのエクスポート

1. **ログをエクスポート**をクリックします
2. 期間を選択します（デフォルト：過去24時間）
3. 形式を選択：**TXT**（人間が読める）または**JSON**（機械が読める）
4. ファイルがダウンロードされます

エクスポートしたログはバグレポートやサポートへの問い合わせ時に役立ちます。

## ログローテーション

ログは自動的にローテーションされます。

| 設定 | デフォルト |
|---|---|
| 最大ログファイルサイズ | 50 MB |
| 保持するローテーションファイル数 | 5 |
| 総最大ログサイズ | 250 MB |

**設定 → システム → ログローテーション**で調整します。古いログファイルはgzipで自動的に圧縮されます。

## ログファイルの場所

ログファイルはサーバーに保存されます：

```
./data/logs/
├── 3dprintforge.log          (アクティブなログ)
├── 3dprintforge.log.1.gz     (ローテーション済み)
├── 3dprintforge.log.2.gz     (ローテーション済み)
└── ...
```

:::tip SSHアクセス
SSH経由でサーバー上のログを直接読むには：
```bash
tail -f ./data/logs/3dprintforge.log
```
:::
