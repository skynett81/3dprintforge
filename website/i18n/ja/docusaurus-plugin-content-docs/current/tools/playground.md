# APIプレイグラウンド

APIプレイグラウンドでは、3DPrintForgeの177のAPIエンドポイントをすべてコードを書かずにブラウザから直接探索・テストできます。

アクセス先：**https://localhost:3443/api/docs**

## APIプレイグラウンドとは

プレイグラウンドはダッシュボードと完全に統合されたインタラクティブなOpenAPIドキュメント（Swagger UI）です。ログイン済みであれば認証済みの状態なので、エンドポイントを直接テストできます。

## ドキュメントのナビゲーション

エンドポイントはカテゴリーに整理されています。

| カテゴリー | エンドポイント数 | 説明 |
|---|---|---|
| プリンター | 24 | ステータス取得、制御、設定 |
| プリント / 履歴 | 18 | 履歴の取得、検索、エクスポート |
| フィラメント | 22 | 在庫、スプール、プロファイル |
| キュー | 12 | プリントキューの管理 |
| 統計 | 15 | 集計統計とエクスポート |
| 通知 | 8 | 通知チャンネルの設定とテスト |
| ユーザー | 10 | ユーザー、ロール、APIキー |
| 設定 | 14 | 設定の読み取りと変更 |
| メンテナンス | 12 | メンテナンスタスクとログ |
| 統合 | 18 | HA、Tibber、Webhookなど |
| ファイルライブラリ | 14 | アップロード、分析、管理 |
| システム | 10 | バックアップ、ヘルス、ログ |

カテゴリーをクリックして展開し、すべてのエンドポイントを表示します。

## エンドポイントのテスト

1. エンドポイントをクリックします（例：`GET /api/printers`）
2. **Try it out**をクリックします
3. パラメーター（フィルター、ページネーション、プリンターIDなど）を入力します
4. **Execute**をクリックします
5. 下にレスポンスを確認：HTTPステータスコード、ヘッダー、JSONボディ

### 例：すべてのプリンターを取得

```
GET /api/printers
```
リアルタイムのステータスを含む登録済みプリンターのリストを返します。

### 例：プリンターにコマンドを送信

```
POST /api/printers/{id}/command
Body: {"command": "pause"}
```

:::warning 本番環境
APIプレイグラウンドは実際のシステムに接続されています。コマンドは実際のプリンターに送信されます。`DELETE`や`POST /command`などの破壊的な操作には注意してください。
:::

## 認証

### セッション認証（ログインユーザー）
ダッシュボードにログイン済みであれば、プレイグラウンドはセッションCookieで既に認証されています。追加設定は不要です。

### APIキー認証

外部アクセスの場合：

1. プレイグラウンド上部の**Authorize**（鍵アイコン）をクリックします
2. **ApiKeyAuth**フィールドにAPIキーを入力：`Bearer あなたのキー`
3. **Authorize**をクリックします

APIキーは**設定 → APIキー**で生成します（[認証](../system/auth)を参照）。

## レート制限

APIにはユーザー/キーごとに**1分あたり200リクエスト**のレート制限があります。プレイグラウンドはレスポンスヘッダー`X-RateLimit-Remaining`で残りのリクエスト数を表示します。

:::info OpenAPI仕様
完全なOpenAPI仕様をYAMLまたはJSONとしてダウンロードします：
- `https://localhost:3443/api/docs/openapi.yaml`
- `https://localhost:3443/api/docs/openapi.json`

仕様を使用してPython、TypeScript、Goなどのクライアントライブラリを生成できます。
:::

## Webhookテスト

Webhook統合を直接テストします。

1. `POST /api/webhooks/test`に移動します
2. ドロップダウンリストからイベントタイプを選択します
3. システムが設定済みのWebhook URLにテストイベントを送信します
4. プレイグラウンドでリクエスト/レスポンスを確認します
