# 출력 대기열

출력 대기열을 사용하면 미리 출력을 계획하고 프린터가 사용 가능해지면 자동으로 전송할 수 있습니다.

이동: **https://localhost:3443/#queue**

## 대기열 만들기

1. 탐색 메뉴에서 **출력 대기열**로 이동합니다
2. **새 작업**(+ 아이콘) 클릭
3. 다음을 입력합니다:
   - **파일 이름** — `.3mf` 또는 `.gcode` 업로드
   - **대상 프린터** — 특정 프린터 선택 또는 **자동**
   - **우선순위** — 낮음 / 보통 / 높음 / 중요
   - **예약 시작** — 지금 또는 특정 날짜/시간
4. **대기열에 추가** 클릭

:::tip 드래그 앤 드롭
파일 탐색기에서 파일을 대기열 페이지로 직접 드래그하여 빠르게 추가할 수 있습니다.
:::

## 파일 추가

### 파일 업로드

1. **업로드** 클릭하거나 업로드 필드에 파일 드래그
2. 지원 형식: `.3mf`, `.gcode`, `.bgcode`
3. 파일이 파일 라이브러리에 저장되고 대기열 작업에 연결됩니다

### 파일 라이브러리에서

1. **파일 라이브러리**로 이동하여 파일 찾기
2. 파일에서 **대기열에 추가** 클릭
3. 기본 설정으로 작업이 생성됩니다 — 필요하면 편집

### 기록에서

1. **기록**에서 이전 프린트 열기
2. **다시 프린트** 클릭
3. 마지막과 같은 설정으로 작업이 추가됩니다

## 우선순위

대기열은 우선순위 순서로 처리됩니다:

| 우선순위 | 색상 | 설명 |
|---|---|---|
| 중요 | 빨간색 | 다른 작업과 관계없이 첫 번째 사용 가능한 프린터에 전송 |
| 높음 | 주황색 | 보통 및 낮은 작업보다 앞에 배치 |
| 보통 | 파란색 | 기본 순서(FIFO) |
| 낮음 | 회색 | 더 높은 작업이 없을 때만 전송 |

같은 우선순위 수준 내에서 수동으로 순서를 변경하려면 대기열에서 작업을 드래그 앤 드롭합니다.

## 자동 디스패치

**자동 디스패치**가 활성화되면 3DPrintForge가 모든 프린터를 모니터링하고 자동으로 다음 작업을 전송합니다:

1. **설정 → 대기열**로 이동합니다
2. **자동 디스패치** 켜기
3. **디스패치 전략** 선택:
   - **첫 번째 사용 가능** — 사용 가능해지는 첫 번째 프린터에 전송
   - **가장 적게 사용됨** — 오늘 가장 적게 프린트한 프린터를 우선
   - **라운드 로빈** — 모든 프린터 간에 균등하게 순환

:::warning 확인
파일이 전송되기 전에 각 디스패치를 수동으로 승인하려면 설정에서 **확인 필요**를 활성화합니다.
:::

## 분산 시작

분산 시작은 모든 프린터가 동시에 시작하고 끝나는 것을 방지하는 데 유용합니다:

1. **새 작업** 대화 상자에서 **고급 설정** 확장
2. **분산 시작** 활성화
3. **프린터 간 지연** 설정(예: 30분)
4. 시스템이 시작 시간을 자동으로 배분합니다

**예시:** 30분 지연이 있는 4개의 동일한 작업은 08:00, 08:30, 09:00, 09:30에 시작됩니다.

## 대기열 상태 및 추적

대기열 개요에는 다음 상태의 모든 작업이 표시됩니다:

| 상태 | 설명 |
|---|---|
| 대기 중 | 작업이 대기열에 있고 프린터를 기다리는 중 |
| 예약됨 | 미래의 예약된 시작 시간이 있음 |
| 전송 중 | 프린터로 전송 중 |
| 프린팅 | 선택한 프린터에서 진행 중 |
| 완료 | 완료 — 기록에 연결됨 |
| 실패 | 전송 또는 프린트 중 오류 |
| 취소됨 | 수동으로 취소됨 |

:::info 알림
작업이 시작, 완료 또는 실패할 때 알림을 받으려면 **설정 → 알림 → 대기열**에서 대기열 이벤트 알림을 활성화합니다. [알림](./notifications)을 참조하세요.
:::
