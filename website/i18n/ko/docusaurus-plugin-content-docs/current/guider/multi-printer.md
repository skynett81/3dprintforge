# 여러 프린터

프린터가 두 대 이상 있나요? Bambu Dashboard는 플리트 관리를 위해 만들어졌습니다 — 한 곳에서 모든 프린터를 모니터링, 제어, 조율할 수 있습니다.

## 새 프린터 추가하기

1. **설정 → 프린터**로 이동합니다
2. **+ 프린터 추가**를 클릭합니다
3. 다음을 입력합니다:

| 필드 | 예시 | 설명 |
|------|------|------|
| 시리얼 번호 (SN) | 01P... | Bambu Handy 또는 프린터 화면에서 찾을 수 있음 |
| IP 주소 | 192.168.1.101 | LAN 모드용 (권장) |
| 액세스 코드 | 12345678 | 프린터 화면의 8자리 코드 |
| 이름 | "Bambu #2 - P1S" | 대시보드에 표시됨 |
| 모델 | P1P, P1S, X1C, A1 | 올바른 아이콘과 기능을 위한 모델 선택 |

4. **연결 테스트**를 클릭합니다 — 녹색 상태가 보여야 합니다
5. **저장**을 클릭합니다

:::tip 프린터에 설명적인 이름 붙이기
"Bambu 1"과 "Bambu 2"는 혼란스럽습니다. "X1C - 생산" 및 "P1S - 프로토타입"과 같은 이름을 사용하여 체계적으로 관리하세요.
:::

## 플리트 개요

모든 프린터가 추가된 후 **플리트** 패널에 모두 표시됩니다. 여기서 확인할 수 있습니다:

```
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│ X1C - 생산      │  │ P1S - 프로토타입 │  │ A1 - 취미방     │
│ ████████░░ 82%  │  │ 유휴            │  │ ████░░░░░░ 38%  │
│ 1시간 24분 남음 │  │ 출력 준비 완료  │  │ 3시간 12분 남음 │
│ 온도: 220/60°C  │  │ AMS: 4 스풀    │  │ 온도: 235/80°C  │
└─────────────────┘  └─────────────────┘  └─────────────────┘
```

다음을 할 수 있습니다:
- 프린터를 클릭하여 전체 상세 뷰 표시
- 모든 온도, AMS 상태, 활성 오류를 한 번에 확인
- 상태별 필터링 (활성 출력, 유휴, 오류)

## 출력 큐 — 작업 분배

출력 큐를 사용하면 한 곳에서 모든 프린터의 출력을 계획할 수 있습니다.

**작동 방식:**
1. **큐**로 이동합니다
2. **+ 작업 추가**를 클릭합니다
3. 파일과 설정을 선택합니다
4. 프린터를 선택하거나 **자동 할당**을 선택합니다

### 자동 할당
자동 할당을 사용하면 대시보드가 다음에 기반하여 프린터를 선택합니다:
- 사용 가능한 용량
- AMS에서 사용 가능한 필라멘트
- 예약된 유지 관리 창

**설정 → 큐 → 자동 할당**에서 활성화합니다.

### 우선순위
큐의 작업을 드래그 앤 드롭하여 순서를 변경합니다. **높은 우선순위** 작업은 일반 작업보다 먼저 실행됩니다.

## 스태거드 스타트 — 전력 급등 방지

많은 프린터를 동시에 시작하면 예열 단계에서 큰 전력 급등이 발생할 수 있습니다. 스태거드 스타트는 시작 시간을 분산합니다:

**활성화 방법:**
1. **설정 → 플리트 → 스태거드 스타트**로 이동합니다
2. **분산 시작**을 활성화합니다
3. 프린터 간 지연을 설정합니다 (권장: 2〜5분)

**3대의 프린터와 3분 지연의 예:**
```
08:00 — 프린터 1이 예열 시작
08:03 — 프린터 2가 예열 시작
08:06 — 프린터 3이 예열 시작
```

:::tip 퓨즈 크기와 관련
X1C는 예열 중 약 1000W를 소비합니다. 3대 동시 = 3000W, 16A 퓨즈가 끊길 수 있습니다. 스태거드 스타트로 문제를 해결합니다.
:::

## 프린터 그룹

프린터 그룹을 통해 프린터를 논리적으로 구성하고 그룹 전체에 명령을 보낼 수 있습니다:

**그룹 만들기:**
1. **설정 → 프린터 그룹**으로 이동합니다
2. **+ 새 그룹**을 클릭합니다
3. 그룹에 이름을 붙입니다 (예: "생산 공간", "취미방")
4. 그룹에 프린터를 추가합니다

**그룹 기능:**
- 그룹의 집계 통계 보기
- 그룹 전체에 동시에 일시 중지 명령 보내기
- 그룹의 유지 관리 창 설정

## 모든 프린터 모니터링

### 멀티 카메라 뷰
**플리트 → 카메라 뷰**로 이동하여 모든 카메라 피드를 나란히 확인합니다:

```
┌──────────────┐  ┌──────────────┐
│  X1C 피드    │  │  P1S 피드    │
│  [라이브]    │  │  [유휴]      │
└──────────────┘  └──────────────┘
┌──────────────┐  ┌──────────────┐
│  A1 피드     │  │  + 추가      │
│  [라이브]    │  │              │
└──────────────┘  └──────────────┘
```

### 프린터별 알림
다른 프린터에 대해 다른 알림 규칙을 구성할 수 있습니다:
- 생산용 프린터: 야간 포함 항상 알림
- 취미용 프린터: 주간에만 알림

설정은 [알림](./varsler-oppsett)을 참조하세요.

## 플리트 운영 팁

- **필라멘트 슬롯 표준화**: 모든 프린터에서 슬롯 1에 PLA 화이트, 슬롯 2에 PLA 블랙 유지 — 작업 배분이 더 쉬워집니다
- **매일 AMS 수준 확인**: 아침 루틴은 [일상적인 사용](./daglig-bruk) 참조
- **순환 유지 관리**: 모든 프린터를 동시에 유지 관리하지 마세요 — 항상 최소 한 대는 활성 상태 유지
- **파일을 명확하게 명명**: `logo_x1c_pla_0.2mm.3mf`와 같은 파일 이름으로 올바른 프린터를 선택하기 쉬워집니다
