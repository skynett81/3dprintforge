# 알림 설정

Bambu Dashboard는 출력 완료부터 중요한 오류까지 모든 것을 Telegram, Discord, 이메일 또는 브라우저 푸시 알림으로 알려줄 수 있습니다.

## 알림 채널 개요

| 채널 | 최적 용도 | 필요한 것 |
|------|-----------|----------|
| Telegram | 빠르게, 어디서든 | Telegram 계정 + 봇 토큰 |
| Discord | 팀/커뮤니티 | Discord 서버 + Webhook URL |
| 이메일 (SMTP) | 공식 알림 | SMTP 서버 |
| 브라우저 푸시 | 데스크톱 알림 | 푸시 지원 브라우저 |

---

## Telegram 봇

### 1단계 — 봇 만들기

1. Telegram을 열고 **@BotFather**를 검색합니다
2. `/newbot`을 보냅니다
3. 봇에 이름을 붙입니다 (예: "Bambu 알림")
4. 봇에 사용자 이름을 붙입니다 (예: `bambu_alrim_bot`) — `bot`으로 끝나야 합니다
5. BotFather가 **API 토큰**을 응답합니다: `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`
6. 이 토큰을 복사하여 저장합니다

### 2단계 — Chat ID 찾기

1. 봇과 대화를 시작합니다 (사용자 이름을 검색하여 **시작**을 클릭)
2. 봇에 메시지를 보냅니다 (예: "안녕")
3. 브라우저에서 `https://api.telegram.org/bot<당신의_토큰>/getUpdates`로 이동합니다
4. `"chat":{"id": 123456789}`를 찾습니다 — 그것이 당신의 Chat ID입니다

### 3단계 — 대시보드에 연결하기

1. **설정 → 알림 → Telegram**으로 이동합니다
2. **봇 토큰**을 붙여넣습니다
3. **Chat ID**를 붙여넣습니다
4. **알림 테스트**를 클릭합니다 — Telegram에서 테스트 메시지를 받아야 합니다
5. **저장**을 클릭합니다

:::tip 그룹 알림
전체 그룹에 알림을 보내고 싶나요? 봇을 Telegram 그룹에 추가하고 그룹 Chat ID (음수, 예: `-100123456789`)를 찾아서 대신 사용하세요.
:::

## Discord Webhook

### 1단계 — Discord에서 Webhook 만들기

1. Discord 서버로 이동합니다
2. 알림을 받고 싶은 채널을 우클릭 → **채널 편집**
3. **연동 → 웹후크**로 이동합니다
4. **새 웹후크**를 클릭합니다
5. 이름을 붙입니다 (예: "Bambu Dashboard")
6. 아바타를 선택합니다 (선택 사항)
7. **웹후크 URL 복사**를 클릭합니다

URL 형식:
```
https://discord.com/api/webhooks/123456789/abcdefghijk...
```

### 2단계 — 대시보드에 입력하기

1. **설정 → 알림 → Discord**로 이동합니다
2. **Webhook URL**을 붙여넣습니다
3. **알림 테스트**를 클릭합니다 — Discord 채널에 테스트 메시지가 와야 합니다
4. **저장**을 클릭합니다

## 이메일 (SMTP)

### 필요한 정보

이메일 제공업체의 SMTP 설정이 필요합니다:

| 제공업체 | SMTP 서버 | 포트 | 암호화 |
|---------|----------|------|--------|
| Gmail | smtp.gmail.com | 587 | TLS |
| Outlook/Hotmail | smtp-mail.outlook.com | 587 | TLS |
| Yahoo | smtp.mail.yahoo.com | 587 | TLS |
| 자체 도메인 | smtp.yourdomain.com | 587 | TLS |

:::warning Gmail은 앱 비밀번호가 필요합니다
Gmail은 일반 비밀번호로 로그인을 차단합니다. Google 계정 → 보안 → 2단계 인증 → 앱 비밀번호에서 **앱 비밀번호**를 만들어야 합니다.
:::

### 대시보드에서 구성

1. **설정 → 알림 → 이메일**로 이동합니다
2. 다음을 입력합니다:
   - **SMTP 서버**: 예: `smtp.gmail.com`
   - **포트**: `587`
   - **사용자 이름**: 이메일 주소
   - **비밀번호**: 앱 비밀번호 또는 일반 비밀번호
   - **발신 주소**: 알림이 발송되는 이메일
   - **수신 주소**: 알림을 받을 이메일
3. **이메일 테스트**를 클릭합니다
4. **저장**을 클릭합니다

## 브라우저 푸시 알림

푸시 알림은 브라우저 탭이 백그라운드에 있을 때도 데스크톱에 시스템 알림으로 표시됩니다.

**활성화:**
1. **설정 → 알림 → 푸시 알림**으로 이동합니다
2. **푸시 알림 활성화**를 클릭합니다
3. 브라우저가 권한을 요청합니다 — **허용**을 클릭합니다
4. **알림 테스트**를 클릭합니다

:::info 활성화한 브라우저에서만 작동
푸시 알림은 특정 브라우저와 장치에 연결됩니다. 알림을 받고 싶은 각 장치에서 활성화하세요.
:::

## 알림 이벤트 선택

알림 채널을 설정한 후 어떤 이벤트가 알림을 트리거할지 정확하게 선택할 수 있습니다:

**설정 → 알림 → 이벤트 아래:**

| 이벤트 | 권장 |
|--------|------|
| 출력 완료 | 예 |
| 출력 실패 / 취소 | 예 |
| Print Guard: 스파게티 감지 | 예 |
| HMS 오류 (심각) | 예 |
| HMS 경고 | 선택 |
| 필라멘트 부족 | 예 |
| AMS 오류 | 예 |
| 프린터 연결 끊김 | 선택 |
| 유지 관리 알림 | 선택 |
| 야간 백업 완료 | 아니요 (소음) |

## 방해 금지 시간 (밤에 알림 받지 않기)

새벽 3시에 출력 완료 알림에 깨어나지 않으려면:

1. **설정 → 알림 → 방해 금지 시간**으로 이동합니다
2. **방해 금지 시간**을 활성화합니다
3. 시작 시간과 종료 시간을 설정합니다 (예: **22:00에서 07:00까지**)
4. 방해 금지 기간 중에도 알림을 받을 이벤트를 선택합니다:
   - **심각한 HMS 오류** — 활성화 상태 유지 권장
   - **Print Guard** — 활성화 상태 유지 권장
   - **출력 완료** — 밤에는 끌 수 있습니다

:::tip 방해 없이 야간 출력
방해 금지 시간을 활성화하여 야간에 출력합니다. Print Guard가 지켜봅니다 — 아침에 요약을 받습니다.
:::
