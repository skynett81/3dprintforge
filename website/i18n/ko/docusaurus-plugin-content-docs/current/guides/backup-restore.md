# 백업 및 복원

3DPrintForge는 모든 데이터를 로컬에 저장합니다 — 출력 기록, 필라멘트 재고, 설정, 사용자 등. 정기적인 백업은 서버 오류 또는 마이그레이션 시 데이터를 잃지 않도록 보장합니다.

## 백업에 포함되는 것은?

| 데이터 | 포함 여부 | 비고 |
|--------|----------|------|
| 출력 기록 | 예 | 모든 로그와 통계 |
| 필라멘트 재고 | 예 | 스풀, 무게, 브랜드 |
| 설정 | 예 | 모든 시스템 설정 |
| 프린터 설정 | 예 | IP 주소, 액세스 코드 |
| 사용자 및 역할 | 예 | 비밀번호는 해시로 저장됨 |
| 알림 설정 | 예 | Telegram 토큰 등 |
| 카메라 이미지 | 선택 | 큰 파일이 될 수 있음 |
| 타임랩스 비디오 | 선택 | 기본적으로 제외됨 |

## 자동 야간 백업

기본적으로 매일 밤 03:00에 자동 백업이 실행됩니다.

**자동 백업 보기 및 구성:**
1. **시스템 → 백업**으로 이동합니다
2. **자동 백업** 아래에서 확인합니다:
   - 마지막으로 성공한 백업과 시간
   - 다음 예약된 백업
   - 저장된 백업 수 (기본값: 7일)

**구성:**
- **시간** — 기본 03:00에서 적합한 시간으로 변경
- **보존 기간** — 백업을 보관하는 일수 (7, 14, 30일)
- **저장 위치** — 로컬 폴더 (기본값) 또는 외부 경로
- **압축** — 기본적으로 활성화 (크기를 60〜80% 줄임)

:::info 백업 파일은 기본적으로 여기에 저장됩니다
```
/path/to/3dprintforge/data/backups/
backup-2025-03-22-030000.tar.gz
backup-2025-03-21-030000.tar.gz
...
```
:::

## 수동 백업

언제든지 백업을 수행합니다:

1. **시스템 → 백업**으로 이동합니다
2. **지금 백업**을 클릭합니다
3. 상태가 **완료**를 표시할 때까지 기다립니다
4. **다운로드**를 클릭하여 백업 파일을 다운로드합니다

**터미널을 통한 대안:**
```bash
cd /sti/til/3dprintforge
node scripts/backup.js
```

백업 파일은 파일 이름에 타임스탬프가 포함되어 `data/backups/`에 저장됩니다.

## 백업에서 복원

:::warning 복원은 기존 데이터를 덮어씁니다
모든 기존 데이터가 백업 파일의 내용으로 대체됩니다. 올바른 파일로 복원하는지 확인하세요.
:::

### 대시보드를 통해

1. **시스템 → 백업**으로 이동합니다
2. **복원**을 클릭합니다
3. 목록에서 백업 파일을 선택하거나 디스크에서 백업 파일을 업로드합니다
4. **지금 복원**을 클릭합니다
5. 복원 후 대시보드가 자동으로 재시작됩니다

### 터미널을 통해

```bash
cd /sti/til/3dprintforge
node scripts/restore.js data/backups/backup-2025-03-22-030000.tar.gz
```

복원 후 대시보드를 재시작합니다:
```bash
sudo systemctl restart 3dprintforge
# 또는
npm start
```

## 설정 내보내기 및 가져오기

설정만 보관하고 싶나요 (모든 기록이 아닌)?

**내보내기:**
1. **시스템 → 설정 → 내보내기**로 이동합니다
2. 포함할 항목을 선택합니다:
   - 프린터 설정
   - 알림 설정
   - 사용자 계정
   - 필라멘트 브랜드 및 프로파일
3. **내보내기**를 클릭합니다 — `.json` 파일을 다운로드합니다

**가져오기:**
1. **시스템 → 설정 → 가져오기**로 이동합니다
2. `.json` 파일을 업로드합니다
3. 가져올 부분을 선택합니다
4. **가져오기**를 클릭합니다

:::tip 새 설치에 유용
내보낸 설정은 새 서버로 이동할 때 편리합니다. 새 설치 후 가져오기하면 모든 것을 처음부터 다시 설정할 필요가 없습니다.
:::

## 새 서버로 마이그레이션

모든 데이터와 함께 3DPrintForge를 새 컴퓨터로 이동하는 방법:

### 1단계 — 이전 서버에서 백업 수행

1. **시스템 → 백업 → 지금 백업**으로 이동합니다
2. 백업 파일을 다운로드합니다
3. 파일을 새 서버에 복사합니다 (USB, scp, 네트워크 공유)

### 2단계 — 새 서버에 설치

```bash
git clone https://github.com/skynett81/3dprintforge.git
cd 3dprintforge
./install.sh
```

설치 가이드를 따르세요. 아무것도 구성할 필요 없습니다 — 대시보드를 실행하기만 하면 됩니다.

### 3단계 — 백업 복원

새 서버에서 대시보드가 실행되면:

1. **시스템 → 백업 → 복원**으로 이동합니다
2. 이전 서버의 백업 파일을 업로드합니다
3. **지금 복원**을 클릭합니다

이제 기록, 필라멘트 재고, 설정, 사용자가 모두 제자리에 있습니다.

### 4단계 — 연결 확인

1. **설정 → 프린터**로 이동합니다
2. 각 프린터에 대한 연결을 테스트합니다
3. IP 주소가 여전히 올바른지 확인합니다 (새 서버는 다른 IP를 가질 수 있습니다)

## 올바른 백업 습관을 위한 팁

- **복원을 테스트하세요** — 백업을 수행하고 테스트 컴퓨터에서 적어도 한 번 복원하세요. 테스트되지 않은 백업은 백업이 아닙니다.
- **외부에 저장하세요** — 정기적으로 백업 파일을 외부 디스크나 클라우드 저장소 (Nextcloud, Google Drive 등)에 복사합니다
- **알림을 설정하세요** — **설정 → 알림 → 이벤트**에서 "백업 실패"에 대한 알림을 활성화하여 문제가 발생하면 즉시 알 수 있도록 합니다
