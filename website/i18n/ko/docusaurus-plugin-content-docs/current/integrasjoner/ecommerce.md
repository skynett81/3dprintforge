# 전자상거래

전자상거래 모듈은 고객, 주문 및 청구를 관리하는 완전한 시스템을 제공합니다 — 전문적 또는 세미 프로로 3D 프린트를 판매하는 분들에게 완벽합니다.

이동: **https://localhost:3443/#orders**

:::danger 전자상거래 라이선스 필요
전자상거래 모듈은 유효한 라이선스가 필요합니다. 라이선스는 **[geektech.no](https://geektech.no)를 통해서만 구매**할 수 있습니다. 활성 라이선스 없이는 모듈이 잠겨 사용할 수 없습니다.
:::

## 라이선스 — 구매 및 활성화

### 라이선스 구매

1. **[geektech.no](https://geektech.no)**에 접속하여 계정 생성
2. **Bambu Dashboard — 전자상거래 라이선스** 선택
3. 라이선스 유형 선택:

| 라이선스 유형 | 설명 | 프린터 수 |
|---|---|---|
| **Hobby** | 프린터 1대, 개인 사용 및 소규모 판매 | 1 |
| **Professional** | 최대 5대, 상업적 사용 | 1–5 |
| **Enterprise** | 무제한 프린터, 완전 지원 | 무제한 |

4. 결제 완료
5. 이메일로 **라이선스 키** 수신

### 라이선스 활성화

1. 대시보드에서 **설정 → 전자상거래**로 이동합니다
2. 필드에 **라이선스 키** 붙여넣기
3. **라이선스 활성화** 클릭
4. 대시보드가 geektech.no 서버에서 키 인증
5. 성공적으로 활성화되면 라이선스 유형, 만료일 및 프린터 수가 표시됩니다

:::warning 라이선스 키는 설치에 연결됩니다
키는 하나의 Bambu Dashboard 설치에 활성화됩니다. 새 서버로 라이선스를 이전해야 하는 경우 [geektech.no](https://geektech.no)에 문의하십시오.
:::

### 라이선스 유효성 검사

- 라이선스는 시작 시 **온라인으로 검증**되며 이후 24시간마다 검증
- 네트워크 오류 시 라이선스는 최대 **7일 오프라인** 작동
- 만료된 라이선스 → 모듈이 잠기지만 기존 데이터는 유지
- 갱신은 **[geektech.no](https://geektech.no)** → 내 라이선스 → 갱신에서 진행

### 라이선스 상태 확인

**설정 → 전자상거래**로 이동하거나 API 호출:

```bash
curl -sk https://localhost:3443/api/ecom-license/status
```

응답에 포함된 정보:
```json
{
  "active": true,
  "type": "professional",
  "expires": "2027-03-22",
  "printers": 5,
  "licensee": "회사명",
  "provider": "geektech.no"
}
```

## 고객

### 고객 만들기

1. **전자상거래 → 고객**으로 이동합니다
2. **새 고객** 클릭
3. 다음을 입력합니다:
   - **이름 / 회사명**
   - **담당자** (기업의 경우)
   - **이메일 주소**
   - **전화번호**
   - **주소** (청구 주소)
   - **사업자등록번호** (선택 사항, 부가세 등록 업체)
   - **메모** — 내부 메모
4. **만들기** 클릭

### 고객 개요

고객 목록에 표시되는 정보:
- 이름 및 연락처 정보
- 총 주문 수
- 총 매출
- 최근 주문 날짜
- 상태 (활성 / 비활성)

고객을 클릭하면 모든 주문 및 청구 기록을 볼 수 있습니다.

## 주문 관리

### 주문 만들기

1. **전자상거래 → 주문**으로 이동합니다
2. **새 주문** 클릭
3. 목록에서 **고객** 선택
4. 주문 항목 추가:
   - 파일 라이브러리에서 파일/모델 선택하거나 자유 텍스트 항목 추가
   - 수량 및 단가 입력
   - 프로젝트에 연결된 경우 시스템이 자동으로 비용 계산
5. **납품 날짜** 입력 (예상)
6. **주문 만들기** 클릭

### 주문 상태

| 상태 | 설명 |
|---|---|
| 문의 | 문의 수신, 미확인 |
| 확인됨 | 고객이 확인 |
| 제작 중 | 프린팅 진행 중 |
| 납품 준비 | 완료, 수령/배송 대기 |
| 납품됨 | 주문 완료 |
| 취소됨 | 고객 또는 판매자가 취소 |

주문을 클릭한 후 **상태 변경**으로 상태를 업데이트합니다.

### 프린트를 주문에 연결

1. 주문 열기
2. **프린트 연결** 클릭
3. 기록에서 프린트 선택 (다중 선택 지원)
4. 프린트 기록에서 비용 데이터가 자동으로 가져와집니다

## 청구

청구에 대한 자세한 문서는 [프로젝트 → 청구](../funksjoner/projects#청구)를 참조하세요.

주문에서 직접 인보이스를 생성할 수 있습니다:

1. 주문 열기
2. **인보이스 생성** 클릭
3. 금액 및 부가세 확인
4. PDF 다운로드 또는 고객 이메일로 전송

### 인보이스 번호 시리즈

**설정 → 전자상거래**에서 인보이스 번호 시리즈 설정:
- **접두사**: 예: `2026-`
- **시작 번호**: 예: `1001`
- 인보이스 번호는 오름차순으로 자동 할당

## 보고 및 세금

### 수수료 보고

시스템이 모든 거래 수수료를 추적합니다:
- **전자상거래 → 수수료**에서 수수료 확인
- 회계 목적으로 수수료를 보고됨으로 표시
- 기간별 수수료 요약 내보내기

### 통계

**전자상거래 → 통계**에서:
- 월별 매출 (막대 그래프)
- 매출 기준 상위 고객
- 가장 많이 판매된 모델/재료
- 평균 주문 규모

회계 시스템을 위해 CSV로 내보내기.
