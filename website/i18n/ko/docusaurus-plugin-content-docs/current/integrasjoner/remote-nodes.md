# 원격 서버 (Remote Nodes)

원격 서버 기능을 사용하면 여러 Bambu Dashboard 인스턴스를 연결하여 동일한 네트워크에 있든 다른 위치에 있든 관계없이 하나의 중앙 인터페이스에서 모든 프린터를 보고 제어할 수 있습니다.

이동: **https://localhost:3443/#settings** → **통합 → 원격 서버**

## 사용 시나리오

- **집 + 사무실** — 동일한 대시보드에서 두 위치의 프린터 확인
- **메이커스페이스** — 방의 모든 인스턴스를 위한 중앙 대시보드
- **게스트 인스턴스** — 전체 액세스 없이 고객에게 제한된 열람 권한 제공

## 아키텍처

```
기본 인스턴스 (내 PC)
  ├── 프린터 A (로컬 MQTT)
  ├── 프린터 B (로컬 MQTT)
  └── 원격 서버: 보조 인스턴스
        ├── 프린터 C (원격 위치의 MQTT)
        └── 프린터 D (원격 위치의 MQTT)
```

기본 인스턴스가 REST API를 통해 원격 서버를 폴링하고 데이터를 로컬에서 집계합니다.

## 원격 서버 추가

### 1단계: 원격 인스턴스에서 API 키 생성

1. 원격 인스턴스에 로그인 (예: `https://192.168.2.50:3443`)
2. **설정 → API 키**로 이동합니다
3. **새 키** 클릭 → 이름을 「기본 노드」로 지정
4. 권한 설정: **읽기** (최소) 또는 **읽기 + 쓰기** (원격 제어용)
5. 키 복사

### 2단계: 기본 인스턴스에서 연결

1. **설정 → 원격 서버**로 이동합니다
2. **원격 서버 추가** 클릭
3. 다음을 입력합니다:
   - **이름**: 예: 「사무실」또는 「차고」
   - **URL**: `https://192.168.2.50:3443` 또는 외부 URL
   - **API 키**: 1단계의 키
4. **연결 테스트** 클릭
5. **저장** 클릭

:::warning 자체 서명 인증서
원격 인스턴스가 자체 서명 인증서를 사용하는 경우 **TLS 오류 무시** 활성화 — 내부 네트워크 연결에만 이 옵션을 사용하세요.
:::

## 집계 보기

연결 후 원격 프린터가 다음에 표시됩니다:

- **플리트 개요** — 원격 서버 이름과 클라우드 아이콘으로 표시
- **통계** — 모든 인스턴스에 걸쳐 집계
- **필라멘트 재고** — 통합 개요

## 원격 제어

**읽기 + 쓰기** 권한으로 원격 프린터를 직접 제어할 수 있습니다:

- 일시 정지 / 재개 / 정지
- 출력 대기열에 추가 (작업이 원격 인스턴스로 전송)
- 카메라 스트림 보기 (원격 인스턴스를 통해 프록시)

:::info 지연 시간
원격 서버를 통한 카메라 스트림은 네트워크 속도와 거리에 따라 눈에 띄는 지연이 있을 수 있습니다.
:::

## 접근 제어

원격 서버가 공유하는 데이터를 제한합니다:

1. 원격 인스턴스에서: **설정 → API 키 → [키 이름]**으로 이동합니다
2. 접근 제한:
   - 특정 프린터만
   - 카메라 스트림 없음
   - 쓰기 금지 (읽기 전용)

## 상태 및 모니터링

각 원격 서버의 상태가 **설정 → 원격 서버**에 표시됩니다:

- **연결됨** — 마지막 폴링 성공
- **연결 끊김** — 원격 서버에 연결할 수 없음
- **인증 오류** — API 키가 유효하지 않거나 만료됨
- **마지막 동기화** — 마지막으로 성공한 데이터 동기화의 타임스탬프
