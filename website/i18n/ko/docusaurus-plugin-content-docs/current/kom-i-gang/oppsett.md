# 초기 설정

대시보드가 처음 실행되면 설정 마법사가 자동으로 열립니다.

## 설정 마법사

마법사는 `https://서버주소:3443/setup`에서 사용할 수 있습니다. 다음 과정을 안내합니다:

1. 관리자 사용자 만들기
2. 프린터 추가
3. 연결 테스트
4. 알림 구성 (선택 사항)

## 프린터 추가

프린터에 연결하려면 세 가지 정보가 필요합니다:

| 항목 | 설명 | 예시 |
|------|-------------|---------|
| IP 주소 | 프린터의 로컬 IP | `192.168.1.100` |
| 일련번호 | 15자리, 프린터 아래에 있음 | `01P09C123456789` |
| Access Code | 8자리, 프린터 네트워크 설정에서 찾을 수 있음 | `12345678` |

### 프린터에서 Access Code 찾기

**X1C / P1S / P1P:**
1. 화면에서 **설정**으로 이동합니다
2. **WLAN** 또는 **LAN** 선택
3. **Access Code** 확인

**A1 / A1 Mini:**
1. 화면을 누르고 **설정** 선택
2. **WLAN**으로 이동합니다
3. **Access Code** 확인

:::tip 고정 IP 주소
라우터에서 프린터에 고정 IP 주소를 설정하세요 (DHCP 예약). 그러면 프린터가 새 IP를 받을 때마다 대시보드를 업데이트할 필요가 없습니다.
:::

## AMS 구성

프린터가 연결되면 AMS 상태가 자동으로 업데이트됩니다. 다음이 가능합니다:

- 각 슬롯에 이름과 색상 지정
- 슬롯을 필라멘트 재고에 연결
- 슬롯별 필라멘트 소비 확인

수동 구성은 **설정 → 프린터 → AMS**로 이동합니다.

## HTTPS 인증서 {#https-인증서}

### 자체 서명 인증서 (기본값)

대시보드가 시작 시 자동으로 자체 서명 인증서를 생성합니다. 브라우저에서 신뢰하려면:

- **Chrome/Edge:** 「고급」 클릭 → 「사이트로 계속 이동」
- **Firefox:** 「고급」 클릭 → 「위험을 수용하고 계속」

### 자체 인증서

인증서 파일을 폴더에 배치하고 `config.json`에서 구성합니다:

```json
{
  "ssl": {
    "cert": "/경로/cert.pem",
    "key": "/경로/key.pem"
  }
}
```

:::info Let's Encrypt
도메인 이름을 사용하고 있나요? Let's Encrypt와 Certbot으로 무료 인증서를 생성하고 `cert`와 `key`를 `/etc/letsencrypt/live/도메인/`의 파일로 지정하세요.
:::

## 환경 변수

모든 설정은 환경 변수로 재정의할 수 있습니다:

| 변수 | 기본값 | 설명 |
|----------|---------|-------------|
| `PORT` | `3000` | HTTP 포트 |
| `HTTPS_PORT` | `3443` | HTTPS 포트 |
| `NODE_ENV` | `production` | 환경 |
| `AUTH_SECRET` | (자동) | JWT 비밀 |

## 다중 프린터 설정

**설정 → 프린터 → 프린터 추가**에서 여러 프린터를 추가할 수 있습니다. 대시보드 상단의 프린터 선택기를 사용하여 프린터 간에 전환합니다.
