# 오류 패턴 분석

오류 패턴 분석은 프린트 및 오류의 과거 데이터를 사용하여 패턴, 원인 및 상관관계를 식별하고 구체적인 개선 제안을 제공합니다.

이동: **https://localhost:3443/#error-analysis**

## 분석 대상

시스템이 다음 데이터 포인트를 분석합니다:

- HMS 오류 코드 및 발생 시간
- 오류 발생 시 필라멘트 유형 및 공급업체
- 오류 발생 시 온도 (노즐, 베드, 챔버)
- 프린트 속도 및 프로필
- 요일 및 시간
- 마지막 유지보수 이후 경과 시간
- 프린터 모델 및 펌웨어 버전

## 상관관계 분석

시스템이 오류와 요소 간의 통계적 상관관계를 찾습니다:

**감지되는 상관관계 예시:**
- 「AMS 막힘 오류의 78%가 X 공급업체의 필라멘트에서 발생」
- 「노즐 막힘은 6시간+ 연속 프린팅 후 3배 더 자주 발생」
- 「챔버 온도 18°C 이하에서 접착 오류 증가」
- 「습도 60% 이상에서 스트링 오류 상관관계 (습도계 연결 시)」

통계적으로 유의미한 상관관계 (p < 0.05)가 상위에 표시됩니다.

:::info 데이터 요구사항
분석은 기록에 최소 50개의 프린트가 있을 때 가장 정확합니다. 적은 프린트의 경우 낮은 신뢰도의 추정치가 표시됩니다.
:::

## 개선 제안

분석을 기반으로 구체적인 제안이 생성됩니다:

| 제안 유형 | 예시 |
|---|---|
| 필라멘트 | 「PA-CF에 다른 공급업체로 교체 — 4개 오류 중 3개가 X공급업체 사용」|
| 온도 | 「PETG의 베드 온도를 5°C 올리면 접착 오류 약 60% 감소 예상」|
| 속도 | 「4시간 후 속도를 80%로 줄이면 노즐 막힘 약 45% 감소 예상」|
| 유지보수 | 「익스트루더 기어 청소 — 마모가 압출 오류의 40%와 상관관계」|
| 캘리브레이션 | 「베드 레벨링 실행 — 지난주 접착 오류 15개 중 12개가 잘못된 캘리브레이션과 상관관계」|

각 제안에 표시되는 정보:
- 예상 효과 (오류 % 감소)
- 신뢰도 (낮음 / 중간 / 높음)
- 단계별 구현 방법
- 관련 문서 링크

## 상태 점수에 미치는 영향

분석은 상태 점수와 연결됩니다 ([진단](./diagnostics) 참조):

- 점수를 가장 많이 낮추는 요소 표시
- 각 제안 구현 시 예상 점수 개선 추정
- 잠재적 점수 개선에 따라 제안 우선순위 지정

## 타임라인 보기

**오류 분석 → 타임라인**으로 이동하면 연대순 개요를 볼 수 있습니다:

1. 프린터 및 기간 선택
2. 오류가 타임라인의 점으로 표시되고, 유형별로 색상 코딩
3. 가로 선이 유지보수 작업을 표시
4. 오류 클러스터 (짧은 시간에 많은 오류)가 빨간색으로 강조 표시

클러스터를 클릭하면 해당 특정 기간에 대한 분석이 열립니다.

## 보고서

오류 분석 PDF 보고서를 생성합니다:

1. **보고서 생성** 클릭
2. 기간 선택 (예: 최근 90일)
3. 내용 선택: 상관관계, 제안, 타임라인, 상태 점수
4. PDF 다운로드 또는 이메일로 전송

프린터가 프로젝트에 연결된 경우 보고서는 프로젝트 아래에 저장됩니다.

:::tip 주간 검토
**설정 → 보고서**에서 자동 주간 이메일 보고서를 설정하면 대시보드를 수동으로 방문하지 않아도 최신 정보를 확인할 수 있습니다. [보고서](../system/reports)를 참조하세요.
:::
