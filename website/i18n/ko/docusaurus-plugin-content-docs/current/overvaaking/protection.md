# Print Guard

Print Guard는 Bambu Dashboard의 실시간 모니터링 시스템입니다. 카메라, 센서 및 프린터 데이터를 지속적으로 모니터링하고 문제가 발생하면 구성 가능한 동작을 실행합니다.

이동: **https://localhost:3443/#protection**

## XCam 이벤트 감지

Bambu Lab 프린터는 AI 카메라가 문제를 감지하면 MQTT를 통해 XCam 이벤트를 전송합니다:

| 이벤트 | 코드 | 심각도 |
|---|---|---|
| 스파게티 감지 | `xcam_spaghetti` | 위험 |
| 플레이트 접착 실패 | `xcam_detach` | 높음 |
| 첫 번째 레이어 오작동 | `xcam_first_layer` | 높음 |
| 스트링 | `xcam_stringing` | 중간 |
| 압출 오류 | `xcam_extrusion` | 높음 |

각 이벤트 유형에 대해 하나 이상의 동작을 구성할 수 있습니다:

- **알림** — 활성 알림 채널을 통해 알림 전송
- **일시 정지** — 수동 확인을 위해 프린트 일시 정지
- **정지** — 즉시 프린트 중단
- **없음** — 이벤트 무시 (여전히 기록됨)

:::danger 기본 동작
기본적으로 XCam 이벤트는 **알림** 및 **일시 정지**로 설정됩니다. AI 감지를 완전히 신뢰하는 경우 **정지**로 변경하세요.
:::

## 센서 모니터링

Print Guard가 센서 데이터를 지속적으로 모니터링하고 이상 발생 시 알림을 보냅니다:

### 온도 이상

1. **Print Guard → 온도**로 이동합니다
2. **목표 온도로부터 최대 편차** 설정 (권장: 노즐 ±5°C, 베드 ±3°C)
3. **이상 발생 시 동작** 선택: 알림 / 일시 정지 / 정지
4. 동작 실행 전 **지연** (초) 설정 — 온도가 안정화될 시간을 줍니다

### 필라멘트 부족

시스템이 스풀의 남은 필라멘트를 계산합니다:

1. **Print Guard → 필라멘트**로 이동합니다
2. 그램 단위로 **최소 한도** 설정 (예: 50g)
3. 동작 선택: **일시 정지 및 알림** (권장) - 수동으로 스풀 교체

### 프린트 정지 감지

예기치 않게 프린트가 정지했을 때 감지합니다 (MQTT 타임아웃, 필라멘트 절단 등):

1. **정지 감지** 활성화
2. **타임아웃** 설정 (권장: 데이터 없이 120초 = 정지)
3. 동작: 항상 알림 — 프린트가 이미 정지되어 있을 수 있음

## 구성

### Print Guard 활성화

1. **설정 → Print Guard**로 이동합니다
2. **Print Guard 활성화** 켜기
3. 모니터링할 프린터 선택
4. **저장** 클릭

### 프린터별 규칙

프린터마다 다른 규칙을 적용할 수 있습니다:

1. Print Guard 개요에서 프린터 클릭
2. **전역 규칙 상속** 끄기
3. 이 프린터에 대한 별도 규칙 구성

## 로그 및 이벤트 기록

모든 Print Guard 이벤트가 기록됩니다:

- **Print Guard → 로그**로 이동합니다
- 프린터, 이벤트 유형, 날짜 및 심각도로 필터링
- 이벤트를 클릭하면 자세한 정보와 수행된 동작 확인
- 로그를 CSV로 내보내기

:::tip 오탐
Print Guard가 불필요한 일시 정지를 발생시키는 경우 **Print Guard → 설정 → 감도**에서 감도를 조정합니다. 「낮음」으로 시작하여 점차 올리세요.
:::
