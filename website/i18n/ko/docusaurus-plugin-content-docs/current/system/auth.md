# 인증

Bambu Dashboard는 역할 기반 접근 제어, API 키 및 TOTP를 통한 선택적 이중 인증 (2FA)을 갖춘 다중 사용자를 지원합니다.

이동: **https://localhost:3443/#settings** → **사용자 및 접근**

## 사용자

### 사용자 만들기

1. **설정 → 사용자**로 이동합니다
2. **새 사용자** 클릭
3. 다음을 입력합니다:
   - **사용자 이름** — 로그인에 사용
   - **이메일 주소**
   - **비밀번호** — 최소 12자 권장
   - **역할** — 아래 역할 참조
4. **만들기** 클릭

새 사용자는 **https://localhost:3443/login**에서 로그인할 수 있습니다.

### 비밀번호 변경

1. **프로필**로 이동합니다 (오른쪽 상단 → 사용자 이름 클릭)
2. **비밀번호 변경** 클릭
3. 현재 비밀번호와 새 비밀번호 입력
4. **저장** 클릭

관리자는 **설정 → 사용자 → [사용자] → 비밀번호 재설정**에서 다른 사람의 비밀번호를 재설정할 수 있습니다.

## 역할

| 역할 | 설명 |
|---|---|
| **관리자** | 전체 접근 — 모든 설정, 사용자 및 기능 |
| **운영자** | 프린터 제어, 모든 것 보기, 시스템 설정 변경 불가 |
| **게스트** | 읽기 전용 — 대시보드, 기록 및 통계 보기 |
| **API 사용자** | API 접근만 — 웹 인터페이스 없음 |

### 사용자 정의 역할

1. **설정 → 역할**로 이동합니다
2. **새 역할** 클릭
3. 개별 권한 선택:
   - 대시보드 / 기록 / 통계 보기
   - 프린터 제어 (일시 정지/정지/시작)
   - 필라멘트 재고 관리
   - 대기열 관리
   - 카메라 스트림 보기
   - 설정 변경
   - 사용자 관리
4. **저장** 클릭

## API 키

API 키는 로그인 없이 프로그래밍 방식의 접근을 제공합니다.

### API 키 만들기

1. **설정 → API 키**로 이동합니다
2. **새 키** 클릭
3. 다음을 입력합니다:
   - **이름** — 설명적인 이름 (예: 「Home Assistant」, 「Python 스크립트」)
   - **만료일** — 선택 사항, 보안을 위해 설정
   - **권한** — 역할 또는 특정 권한 선택
4. **생성** 클릭
5. **지금 키를 복사** — 한 번만 표시됩니다

### API 키 사용

모든 API 호출의 HTTP 헤더에 추가합니다:
```
Authorization: Bearer API_키
```

테스트는 [API 플레이그라운드](../verktoy/playground)를 참조하세요.

:::danger 안전한 보관
API 키는 연결된 사용자와 동일한 접근 권한을 갖습니다. 안전하게 보관하고 정기적으로 교체하세요.
:::

## TOTP 2FA

인증 앱 (Google Authenticator, Authy, Bitwarden 등)을 사용하여 이중 인증을 활성화합니다:

### 2FA 활성화

1. **프로필 → 보안 → 이중 인증**으로 이동합니다
2. **2FA 활성화** 클릭
3. 인증 앱으로 QR 코드 스캔
4. 생성된 6자리 코드를 입력하여 확인
5. **복구 코드** (1회용 코드 10개)를 안전한 곳에 저장
6. **활성화** 클릭

### 2FA로 로그인

1. 평소와 같이 사용자 이름과 비밀번호 입력
2. 앱에서 6자리 TOTP 코드 입력
3. **로그인** 클릭

### 모든 사용자에게 2FA 강제

관리자는 모든 사용자에게 2FA를 요구할 수 있습니다:

1. **설정 → 보안 → 2FA 강제**로 이동합니다
2. 설정 활성화
3. 2FA가 없는 사용자는 다음 로그인 시 설정하도록 강제됩니다

## 세션 관리

- 기본 세션 지속 시간: 24시간
- **설정 → 보안 → 세션 기간**에서 조정
- 사용자별 활성 세션 보기 및 개별 세션 종료
