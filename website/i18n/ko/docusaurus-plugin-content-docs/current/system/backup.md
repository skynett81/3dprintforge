# 백업

Bambu Dashboard는 모든 구성, 기록 및 데이터를 백업하여 시스템 오류, 서버 이전 또는 업데이트 문제 발생 시 쉽게 복원할 수 있습니다.

이동: **https://localhost:3443/#settings** → **시스템 → 백업**

## 백업에 포함되는 항목

| 데이터 유형 | 포함 | 비고 |
|---|---|---|
| 프린터 설정 및 구성 | ✅ | |
| 프린트 기록 | ✅ | |
| 필라멘트 재고 | ✅ | |
| 사용자 및 역할 | ✅ | 비밀번호는 해시로 저장 |
| 설정 | ✅ | 알림 구성 포함 |
| 유지보수 로그 | ✅ | |
| 프로젝트 및 인보이스 | ✅ | |
| 파일 라이브러리 (메타데이터) | ✅ | |
| 파일 라이브러리 (파일) | 선택 사항 | 클 수 있음 |
| 타임랩스 영상 | 선택 사항 | 매우 클 수 있음 |
| 갤러리 이미지 | 선택 사항 | |

## 수동 백업 만들기

1. **설정 → 백업**으로 이동합니다
2. 포함할 항목 선택 (위 표 참조)
3. **지금 백업 만들기** 클릭
4. 백업이 생성되는 동안 진행 표시기 표시
5. 백업이 완료되면 **다운로드** 클릭

백업은 파일명에 타임스탬프가 있는 `.zip` 파일로 저장됩니다:
```
bambu-dashboard-backup-2026-03-22T14-30-00.zip
```

## 백업 다운로드

백업 파일은 서버의 백업 폴더에 저장됩니다 (구성 가능). 직접 다운로드도 가능합니다:

1. **백업 → 기존 백업**으로 이동합니다
2. 목록에서 백업 찾기 (날짜순 정렬)
3. **다운로드** (다운로드 아이콘) 클릭

:::info 저장 폴더
기본 저장 폴더: `./data/backups/`. **설정 → 백업 → 저장 폴더**에서 변경합니다.
:::

## 예약된 자동 백업

1. **백업 → 예약**에서 **자동 백업** 활성화
2. 간격 선택:
   - **매일** — 오전 3시 실행 (구성 가능)
   - **매주** — 특정 요일 및 시간
   - **매월** — 월의 첫 날
3. **보관할 백업 수** 선택 (예: 7 — 오래된 것은 자동 삭제)
4. **저장** 클릭

:::tip 외부 저장소
중요 데이터의 경우: 외부 디스크나 네트워크 디스크를 백업 저장 폴더로 마운트하세요. 그러면 시스템 디스크가 실패해도 백업이 살아남습니다.
:::

## 백업에서 복원

:::warning 복원은 기존 데이터를 덮어씁니다
복원은 백업 파일의 내용으로 모든 기존 데이터를 대체합니다. 먼저 현재 데이터의 최신 백업이 있는지 확인하세요.
:::

### 서버의 기존 백업에서

1. **백업 → 기존 백업**으로 이동합니다
2. 목록에서 백업 찾기
3. **복원** 클릭
4. 대화 상자에서 확인
5. 복원 후 시스템이 자동으로 다시 시작됩니다

### 다운로드한 백업 파일에서

1. **백업 업로드** 클릭
2. 컴퓨터에서 `.zip` 파일 선택
3. 파일이 유효성 검사됨 — 포함된 내용 확인
4. **파일에서 복원** 클릭
5. 대화 상자에서 확인

## 백업 유효성 검사

Bambu Dashboard는 복원 전에 모든 백업 파일을 검증합니다:

- ZIP 형식이 유효한지 확인
- 데이터베이스 스키마가 현재 버전과 호환되는지 확인
- 이전 버전의 백업인 경우 경고 표시 (마이그레이션이 자동으로 수행됨)
