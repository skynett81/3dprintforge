# 서버 로그

서버 로그는 3DPrintForge 내부에서 일어나는 일에 대한 통찰력을 제공합니다 — 문제 해결, 모니터링 및 진단에 유용합니다.

이동: **https://localhost:3443/#logs**

## 실시간 보기

로그 스트림이 WebSocket을 통해 실시간으로 업데이트됩니다:

1. **시스템 → 서버 로그**로 이동합니다
2. 새 로그 라인이 자동으로 하단에 표시됩니다
3. **하단 잠금** 클릭으로 항상 최신 로그로 스크롤
4. **동결** 클릭으로 자동 스크롤을 중지하고 기존 라인 읽기

기본 보기에는 최근 500개의 로그 라인이 표시됩니다.

## 로그 수준

각 로그 라인에는 수준이 있습니다:

| 수준 | 색상 | 설명 |
|---|---|---|
| **ERROR** | 빨간색 | 기능에 영향을 미치는 오류 |
| **WARN** | 주황색 | 경고 — 문제가 발생할 수 있음 |
| **INFO** | 파란색 | 정상 운영 정보 |
| **DEBUG** | 회색 | 자세한 개발자 정보 |

:::info 로그 수준 구성
**설정 → 시스템 → 로그 수준**에서 로그 수준을 변경합니다. 정상 운영에는 **INFO**를 사용하세요. **DEBUG**는 많은 데이터를 생성하므로 문제 해결 시에만 사용하세요.
:::

## 필터링

로그 보기 상단의 필터 툴바를 사용합니다:

1. **로그 수준** — ERROR / WARN / INFO / DEBUG 또는 조합만 표시
2. **모듈** — 시스템 모듈로 필터링:
   - `mqtt` — 프린터와의 MQTT 통신
   - `api` — API 요청
   - `db` — 데이터베이스 작업
   - `auth` — 인증 이벤트
   - `queue` — 출력 대기열 이벤트
   - `guard` — Print Guard 이벤트
   - `backup` — 백업 작업
3. **자유 텍스트** — 로그 텍스트 검색 (정규식 지원)
4. **타임스탬프** — 날짜 범위로 필터링

정밀한 문제 해결을 위해 필터를 조합하세요.

## 일반적인 오류 상황

### MQTT 연결 문제

`mqtt` 모듈의 로그 라인을 확인합니다:

```
ERROR [mqtt] 프린터 XXXX 연결 실패: Connection refused
```

**해결책:** 프린터가 켜져 있는지, 액세스 코드가 올바른지, 네트워크가 작동하는지 확인합니다.

### 데이터베이스 오류

```
ERROR [db] 마이그레이션 v95 실패: SQLITE_CONSTRAINT
```

**해결책:** 백업을 만들고 **설정 → 시스템 → 데이터베이스 복구**에서 데이터베이스 복구를 실행합니다.

### 인증 오류

```
WARN [auth] IP 192.168.1.x에서 admin 사용자에 대한 로그인 실패
```

많은 로그인 실패는 브루트 포스 시도를 나타낼 수 있습니다. IP 화이트리스트 활성화를 고려하세요.

## 로그 내보내기

1. **로그 내보내기** 클릭
2. 기간 선택 (기본값: 최근 24시간)
3. 형식 선택: **TXT** (사람이 읽기 쉬운) 또는 **JSON** (기계가 읽기 쉬운)
4. 파일 다운로드

내보낸 로그는 버그 보고 또는 지원팀에 문의할 때 유용합니다.

## 로그 순환

로그가 자동으로 순환됩니다:

| 설정 | 기본값 |
|---|---|
| 최대 로그 파일 크기 | 50 MB |
| 보관할 순환된 파일 수 | 5 |
| 총 최대 로그 크기 | 250 MB |

**설정 → 시스템 → 로그 순환**에서 조정합니다. 오래된 로그 파일은 자동으로 gzip으로 압축됩니다.

## 로그 파일 위치

로그 파일은 서버에 저장됩니다:

```
./data/logs/
├── 3dprintforge.log          (활성 로그)
├── 3dprintforge.log.1.gz     (순환됨)
├── 3dprintforge.log.2.gz     (순환됨)
└── ...
```

:::tip SSH 접근
SSH를 통해 서버에서 직접 로그를 읽으려면:
```bash
tail -f ./data/logs/3dprintforge.log
```
:::
