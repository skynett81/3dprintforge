# API 플레이그라운드

API 플레이그라운드를 사용하면 코드를 작성하지 않고도 브라우저에서 직접 Bambu Dashboard의 177개 모든 API 엔드포인트를 탐색하고 테스트할 수 있습니다.

이동: **https://localhost:3443/api/docs**

## API 플레이그라운드란?

플레이그라운드는 대시보드와 완전히 통합된 OpenAPI 문서 (Swagger UI)의 대화형 버전입니다. 로그인되어 있으면 이미 인증되어 있으므로 엔드포인트를 직접 테스트할 수 있습니다.

## 문서 탐색

엔드포인트가 분류별로 정리되어 있습니다:

| 분류 | 엔드포인트 수 | 설명 |
|---|---|---|
| 프린터 | 24 | 상태 조회, 제어, 구성 |
| 프린트 / 기록 | 18 | 기록 조회, 검색, 내보내기 |
| 필라멘트 | 22 | 재고, 스풀, 프로필 |
| 대기열 | 12 | 출력 대기열 관리 |
| 통계 | 15 | 집계 통계 및 내보내기 |
| 알림 | 8 | 알림 채널 구성 및 테스트 |
| 사용자 | 10 | 사용자, 역할, API 키 |
| 설정 | 14 | 구성 읽기 및 변경 |
| 유지보수 | 12 | 유지보수 작업 및 로그 |
| 통합 | 18 | HA, Tibber, 웹훅 등 |
| 파일 라이브러리 | 14 | 업로드, 분석, 관리 |
| 시스템 | 10 | 백업, 상태, 로그 |

분류를 클릭하면 모든 엔드포인트가 펼쳐집니다.

## 엔드포인트 테스트

1. 엔드포인트 클릭 (예: `GET /api/printers`)
2. **Try it out** (사용해 보기) 클릭
3. 파라미터 입력 (필터, 페이지네이션, 프린터 ID 등)
4. **Execute** 클릭
5. 응답 확인: HTTP 상태 코드, 헤더 및 JSON 본문

### 예시: 모든 프린터 조회

```
GET /api/printers
```
실시간 상태를 포함한 등록된 모든 프린터 목록을 반환합니다.

### 예시: 프린터에 명령 전송

```
POST /api/printers/{id}/command
Body: {"command": "pause"}
```

:::warning 프로덕션 환경
API 플레이그라운드는 실제 시스템에 연결되어 있습니다. 명령이 실제 프린터로 전송됩니다. `DELETE` 및 `POST /command`와 같은 파괴적인 작업에 주의하세요.
:::

## 인증

### 세션 인증 (로그인한 사용자)
대시보드에 로그인되어 있으면 플레이그라운드가 세션 쿠키를 통해 이미 인증되어 있습니다. 추가 구성이 필요하지 않습니다.

### API 키 인증

외부 접근의 경우:

1. 플레이그라운드 상단의 **Authorize** (자물쇠 아이콘) 클릭
2. **ApiKeyAuth** 필드에 API 키 입력: `Bearer API_키`
3. **Authorize** 클릭

**설정 → API 키**에서 API 키를 생성합니다 ([인증](../system/auth) 참조).

## 속도 제한

API는 사용자/키당 **분당 200개 요청**의 속도 제한이 있습니다. 플레이그라운드는 응답 헤더 `X-RateLimit-Remaining`에 남은 요청 수를 표시합니다.

:::info OpenAPI 사양
전체 OpenAPI 사양을 YAML 또는 JSON으로 다운로드합니다:
- `https://localhost:3443/api/docs/openapi.yaml`
- `https://localhost:3443/api/docs/openapi.json`

사양을 사용하여 Python, TypeScript, Go 등에서 클라이언트 라이브러리를 생성하세요.
:::

## 웹훅 테스트

웹훅 통합을 직접 테스트합니다:

1. `POST /api/webhooks/test`로 이동합니다
2. 드롭다운에서 이벤트 유형 선택
3. 시스템이 구성된 웹훅 URL로 테스트 이벤트를 전송합니다
4. 플레이그라운드에서 요청/응답 확인
