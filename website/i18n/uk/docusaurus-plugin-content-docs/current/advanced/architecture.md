# Технічна архітектура

## Системна діаграма

```
Browser <──WebSocket──> Node.js <──MQTTS:8883──> Printer
Browser <──WS:9001+──> ffmpeg  <──RTSPS:322───> Camera
```

Дашборд спілкується з принтером через MQTT по TLS (порт 8883) та з камерою через RTSPS (порт 322). Браузер підключається до дашборду через HTTPS та WebSocket.

## Технічний стек

| Шар | Технологія |
|-----|-----------|
| Frontend | Vanilla HTML/CSS/JS — 76 компонентних модулів, без кроків збірки, без фреймворків |
| Backend | Node.js 22 з 3 npm-пакетами: `mqtt`, `ws`, `basic-ftp` |
| База даних | SQLite (вбудована в Node.js 22 через `--experimental-sqlite`) |
| Камера | ffmpeg транскодує RTSPS до MPEG1, jsmpeg рендерить у браузері |
| Реальний час | WebSocket-хаб надсилає стан принтера всім підключеним клієнтам |
| Протокол | MQTT по TLS (порт 8883) з LAN Access Code принтера |

## Порти

| Порт | Протокол | Напрямок | Опис |
|------|-----------|---------|-------------|
| 3000 | HTTP + WS | Вхідний | Дашборд (перенаправляє на HTTPS) |
| 3443 | HTTPS + WSS | Вхідний | Захищений дашборд (стандарт) |
| 9001+ | WS | Вхідний | Потоки камери (один на принтер) |
| 8883 | MQTTS | Вихідний | Підключення до принтера |
| 322 | RTSPS | Вихідний | Камера з принтера |

## База даних

SQLite-база даних вбудована в Node.js 22 і не вимагає зовнішньої установки. Схема обробляється 105 міграціями в `db/migrations.js`.

Основні таблиці:

- `printers` — конфігурація принтера
- `print_history` — всі завдання друку
- `filaments` — сховище філаменту
- `ams_slots` — прив'язка слотів AMS
- `queue` — черга друку
- `notifications_config` — налаштування сповіщень
- `maintenance_log` — журнал обслуговування

## Безпека

- HTTPS з автоматично згенерованим сертифікатом (або власним)
- Автентифікація на основі JWT
- CSP та HSTS-заголовки
- Обмеження запитів (200 запитів/хв)
- Відсутність зовнішньої хмарної залежності для основних функцій
