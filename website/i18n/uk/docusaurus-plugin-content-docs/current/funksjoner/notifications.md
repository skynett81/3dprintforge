# Сповіщення

Bambu Dashboard підтримує сповіщення через численні канали, щоб ви завжди знали, що відбувається з вашими принтерами — вдома чи в дорозі.

Перейдіть до: **https://localhost:3443/#settings** → вкладка **Сповіщення**

## Доступні канали

| Канал | Потрібно | Підтримує зображення |
|-------|---------|----------------------|
| Telegram | Токен бота + Chat-ID | ✅ |
| Discord | URL вебхука | ✅ |
| Електронна пошта | SMTP-сервер | ✅ |
| Webhook | URL + необов'язковий ключ | ✅ (base64) |
| ntfy | сервер ntfy + тема | ❌ |
| Pushover | API-токен + ключ користувача | ✅ |
| SMS (Twilio) | Account SID + Auth token | ❌ |
| Push-сповіщення браузера | Конфігурація не потрібна | ❌ |

## Налаштування для кожного каналу

### Telegram

1. Створіть бота через [@BotFather](https://t.me/BotFather) — надішліть `/newbot`
2. Скопіюйте **токен бота** (формат: `123456789:ABC-def...`)
3. Почніть розмову з ботом і надішліть `/start`
4. Знайдіть свій **Chat-ID**: перейдіть до `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
5. У Bambu Dashboard: вставте токен та Chat-ID, натисніть **Тест**

:::tip Груповий канал
Ви можете використовувати групу Telegram як отримувача. Chat-ID для груп починається з `-`.
:::

### Discord

1. Відкрийте сервер Discord, до якого хочете надсилати сповіщення
2. Перейдіть до налаштувань каналу → **Інтеграції → Вебхуки**
3. Натисніть **Новий вебхук**, дайте назву та виберіть канал
4. Скопіюйте URL вебхука
5. Вставте URL у Bambu Dashboard і натисніть **Тест**

### Електронна пошта

1. Заповніть SMTP-сервер, порт (зазвичай 587 для TLS)
2. Ім'я користувача та пароль для облікового запису SMTP
3. Адреса **Від** та адреса(и) **До** (розділені комами для кількох)
4. Активуйте **TLS/STARTTLS** для безпечного надсилання
5. Натисніть **Тест** для відправки тестового електронного листа

:::warning Gmail
Використовуйте **Пароль програми** для Gmail, а не звичайний пароль. Спочатку активуйте двофакторну аутентифікацію у своєму обліковому записі Google.
:::

### ntfy

1. Створіть тему на [ntfy.sh](https://ntfy.sh) або запустіть власний сервер ntfy
2. Заповніть URL сервера (наприклад, `https://ntfy.sh`) та назву теми
3. Встановіть програму ntfy на телефон і підпишіться на ту ж тему
4. Натисніть **Тест**

### Pushover

1. Створіть обліковий запис на [pushover.net](https://pushover.net)
2. Створіть нову програму — скопіюйте **API Token**
3. Знайдіть свій **User Key** на дашборді Pushover
4. Заповніть обидва поля в Bambu Dashboard і натисніть **Тест**

### Webhook (власний)

Bambu Dashboard надсилає HTTP POST з JSON-навантаженням:

```json
{
  "event": "print_complete",
  "printer": "Мій X1C",
  "printer_id": "abc123",
  "timestamp": "2026-03-22T14:30:00Z",
  "data": {
    "file": "benchy.3mf",
    "duration_minutes": 47,
    "filament_used_g": 12.4
  }
}
```

Додайте **Секретний ключ** для перевірки запитів підписом HMAC-SHA256 у заголовку `X-Bambu-Signature`.

## Фільтр подій

Виберіть, які події повинні ініціювати сповіщення для кожного каналу:

| Подія | Опис |
|-------|------|
| Друк розпочато | Починається новий друк |
| Друк завершено | Друк готовий (із зображенням) |
| Друк не вдався | Друк перервано через помилку |
| Друк призупинено | Ручна або автоматична пауза |
| Print Guard спрацював | XCam або датчик ініціювали дію |
| Малий залишок філаменту | Котушка майже порожня |
| Помилка AMS | Блокування, вологий філамент тощо |
| Принтер відключений | Втрачено MQTT-підключення |
| Завдання черги відправлено | Завдання відправлено з черги |

Позначте потрібні події для кожного каналу окремо.

## Режим тиші

Уникайте сповіщень вночі:

1. Активуйте **Режим тиші** в налаштуваннях сповіщень
2. Встановіть **З** та **До** (наприклад, 23:00 → 07:00)
3. Виберіть **Часовий пояс** для таймера
4. Критичні сповіщення (помилки Print Guard) можна обійти — позначте **Завжди надсилати критичні**

## Push-сповіщення браузера

Отримуйте сповіщення безпосередньо в браузері без програми:

1. Перейдіть до **Налаштування → Сповіщення → Push-сповіщення браузера**
2. Натисніть **Активувати push-сповіщення**
3. Прийміть діалог дозволу від браузера
4. Сповіщення працюють, навіть якщо дашборд згорнутий (потребує відкритої вкладки)

:::info PWA
Встановіть Bambu Dashboard як PWA для push-сповіщень у фоновому режимі без відкритої вкладки. Дивіться [PWA](../system/pwa).
:::
