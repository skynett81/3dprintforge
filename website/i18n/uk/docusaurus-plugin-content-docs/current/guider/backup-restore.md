# Резервне копіювання та відновлення

Bambu Dashboard зберігає всі дані локально — історія друку, сховище філаменту, налаштування, користувачі тощо. Регулярне резервне копіювання гарантує, що ви нічого не втратите при збої сервера або переїзді.

## Що включено до резервної копії?

| Дані | Включено | Примітка |
|------|---------|---------|
| Історія друку | Так | Всі журнали та статистика |
| Сховище філаменту | Так | Котушки, ваги, марки |
| Налаштування | Так | Всі системні налаштування |
| Конфігурація принтерів | Так | IP-адреси, коди доступу |
| Користувачі та ролі | Так | Паролі зберігаються як хеш |
| Конфігурація сповіщень | Так | Токени Telegram тощо |
| Знімки камери | Необов'язково | Можуть бути великими файлами |
| Відео time-lapse | Необов'язково | Виключено за замовчуванням |

## Автоматичне нічне резервне копіювання

За замовчуванням автоматичне резервне копіювання виконується щоночі о 03:00.

**Перегляд та налаштування автоматичного резервного копіювання:**
1. Перейдіть до **Система → Резервне копіювання**
2. У розділі **Автоматичне резервне копіювання** ви бачите:
   - Останнє успішне резервне копіювання та час
   - Наступне заплановане резервне копіювання
   - Кількість збережених резервних копій (за замовчуванням: 7 днів)

**Налаштування:**
- **Час** — змініть з типового 03:00 на зручний для вас час
- **Термін зберігання** — кількість днів зберігання резервних копій (7, 14, 30 днів)
- **Місце зберігання** — локальна папка (за замовчуванням) або зовнішній шлях
- **Стиснення** — увімкнено за замовчуванням (зменшує розмір на 60–80%)

:::info Файли резервних копій зберігаються тут за замовчуванням
```
/шлях/до/bambu-dashboard/data/backups/
backup-2025-03-22-030000.tar.gz
backup-2025-03-21-030000.tar.gz
...
```
:::

## Ручне резервне копіювання

Створіть резервну копію в будь-який час:

1. Перейдіть до **Система → Резервне копіювання**
2. Натисніть **Створити резервну копію зараз**
3. Зачекайте, доки статус покаже **Завершено**
4. Завантажте файл резервної копії, натиснувши **Завантажити**

**Альтернативно через термінал:**
```bash
cd /шлях/до/bambu-dashboard
node scripts/backup.js
```

Файл резервної копії зберігається в `data/backups/` із часовою міткою в назві файлу.

## Відновлення з резервної копії

:::warning Відновлення перезаписує існуючі дані
Всі існуючі дані замінюються вмістом файлу резервної копії. Переконайтеся, що відновлюєте правильний файл.
:::

### Через панель управління

1. Перейдіть до **Система → Резервне копіювання**
2. Натисніть **Відновити**
3. Виберіть файл резервної копії зі списку або завантажте файл резервної копії з диска
4. Натисніть **Відновити зараз**
5. Панель управління автоматично перезапускається після відновлення

### Через термінал

```bash
cd /шлях/до/bambu-dashboard
node scripts/restore.js data/backups/backup-2025-03-22-030000.tar.gz
```

Після відновлення перезапустіть панель управління:
```bash
sudo systemctl restart bambu-dashboard
# або
npm start
```

## Експорт та імпорт налаштувань

Хочете зберегти лише налаштування (не всю історію)?

**Експорт:**
1. Перейдіть до **Система → Налаштування → Експорт**
2. Виберіть, що включити:
   - Конфігурацію принтерів
   - Конфігурацію сповіщень
   - Облікові записи користувачів
   - Марки та профілі філаментів
3. Натисніть **Експортувати** — завантажується файл `.json`

**Імпорт:**
1. Перейдіть до **Система → Налаштування → Імпорт**
2. Завантажте файл `.json`
3. Виберіть, які частини імпортувати
4. Натисніть **Імпортувати**

:::tip Корисно при новій установці
Експортовані налаштування зручно взяти з собою на новий сервер. Імпортуйте їх після нової установки, щоб не налаштовувати все заново.
:::

## Перенесення на новий сервер

Як перенести Bambu Dashboard з усіма даними на нову машину:

### Крок 1 — Створіть резервну копію на старому сервері

1. Перейдіть до **Система → Резервне копіювання → Створити резервну копію зараз**
2. Завантажте файл резервної копії
3. Скопіюйте файл на новий сервер (USB, scp, мережева папка)

### Крок 2 — Встановіть на новому сервері

```bash
git clone https://github.com/skynett81/bambu-dashboard.git
cd bambu-dashboard
./install.sh
```

Дотримуйтесь посібника зі встановлення. Нічого не потрібно налаштовувати — просто запустіть панель управління.

### Крок 3 — Відновіть резервну копію

Коли панель управління запущена на новому сервері:

1. Перейдіть до **Система → Резервне копіювання → Відновити**
2. Завантажте файл резервної копії зі старого сервера
3. Натисніть **Відновити зараз**

Все тепер на місці: історія, сховище філаменту, налаштування та користувачі.

### Крок 4 — Перевірте з'єднання

1. Перейдіть до **Налаштування → Принтери**
2. Перевірте з'єднання з кожним принтером
3. Переконайтеся, що IP-адреси все ще правильні (новий сервер може мати іншу IP)

## Поради щодо гарних практик резервного копіювання

- **Перевіряйте відновлення** — створіть резервну копію та відновіть на тестовій машині хоча б один раз. Неперевірені резервні копії — не резервні копії.
- **Зберігайте зовні** — регулярно копіюйте файл резервної копії на зовнішній диск або хмарне сховище (Nextcloud, Google Drive тощо)
- **Налаштуйте сповіщення** — увімкніть сповіщення "Резервне копіювання не вдалося" у **Налаштування → Сповіщення → Події**, щоб одразу знати, коли щось пішло не так
