# Налаштування сповіщень

Bambu Dashboard може сповіщати вас про все — від завершених друків до критичних помилок — через Telegram, Discord, електронну пошту або push-сповіщення браузера.

## Огляд каналів сповіщень

| Канал | Найкраще для | Вимагає |
|-------|-------------|---------|
| Telegram | Швидко, будь-де | Обліковий запис Telegram + токен бота |
| Discord | Команда/спільнота | Сервер Discord + URL вебхука |
| Електронна пошта (SMTP) | Офіційні сповіщення | SMTP-сервер |
| Push браузера | Сповіщення на робочому столі | Браузер із підтримкою push |

## Бот Telegram

### Крок 1 — Створіть бота

1. Відкрийте Telegram і знайдіть **@BotFather**
2. Надішліть `/newbot`
3. Дайте боту ім'я (наприклад, "Bambu Сповіщення")
4. Дайте боту ім'я користувача (наприклад, `bambu_spovishchennia_bot`) — має закінчуватися на `bot`
5. BotFather відповість **API-токеном**: `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`
6. Скопіюйте та збережіть цей токен

### Крок 2 — Знайдіть ваш Chat ID

1. Почніть розмову з вашим ботом (знайдіть ім'я користувача та натисніть **Start**)
2. Надішліть повідомлення боту (наприклад, "привіт")
3. Перейдіть на `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates` у браузері
4. Знайдіть `"chat":{"id": 123456789}` — це ваш Chat ID

### Крок 3 — Підключіть до панелі управління

1. Перейдіть до **Налаштування → Сповіщення → Telegram**
2. Вставте **Токен бота**
3. Вставте **Chat ID**
4. Натисніть **Перевірити сповіщення** — ви повинні отримати тестове повідомлення в Telegram
5. Натисніть **Зберегти**

:::tip Групове сповіщення
Хочете сповіщати цілу групу? Додайте бота до групи Telegram, знайдіть груповий Chat ID (від'ємне число, наприклад `-100123456789`) і використайте його замість особистого.
:::

## Вебхук Discord

### Крок 1 — Створіть вебхук у Discord

1. Перейдіть до свого сервера Discord
2. Клацніть правою кнопкою миші на канал, де хочете отримувати сповіщення → **Редагувати канал**
3. Перейдіть до **Інтеграції → Вебхуки**
4. Натисніть **Новий вебхук**
5. Дайте йому ім'я (наприклад, "Bambu Dashboard")
6. Виберіть аватар (необов'язково)
7. Натисніть **Скопіювати URL вебхука**

URL виглядає так:
```
https://discord.com/api/webhooks/123456789/abcdefghijk...
```

### Крок 2 — Додайте до панелі управління

1. Перейдіть до **Налаштування → Сповіщення → Discord**
2. Вставте **URL вебхука**
3. Натисніть **Перевірити сповіщення** — канал Discord має отримати тестове повідомлення
4. Натисніть **Зберегти**

## Електронна пошта (SMTP)

### Необхідна інформація

Вам потрібні налаштування SMTP від вашого поштового провайдера:

| Провайдер | SMTP-сервер | Порт | Шифрування |
|----------|------------|------|------------|
| Gmail | smtp.gmail.com | 587 | TLS |
| Outlook/Hotmail | smtp-mail.outlook.com | 587 | TLS |
| Yahoo | smtp.mail.yahoo.com | 587 | TLS |
| Власний домен | smtp.вашдомен.ua | 587 | TLS |

:::warning Gmail вимагає пароль програми
Gmail блокує вхід зі звичайним паролем. Потрібно створити **Пароль програми** в обліковому записі Google → Безпека → Двоетапна перевірка → Паролі програм.
:::

### Налаштування в панелі управління

1. Перейдіть до **Налаштування → Сповіщення → Електронна пошта**
2. Заповніть:
   - **SMTP-сервер**: наприклад `smtp.gmail.com`
   - **Порт**: `587`
   - **Ім'я користувача**: ваша адреса електронної пошти
   - **Пароль**: пароль програми або звичайний пароль
   - **Адреса "Від"**: електронна пошта, з якої надсилається сповіщення
   - **Адреса "Кому"**: електронна пошта, на яку ви хочете отримувати сповіщення
3. Натисніть **Перевірити email**
4. Натисніть **Зберегти**

## Push-сповіщення браузера

Push-сповіщення з'являються як системні сповіщення на робочому столі — навіть коли вкладка браузера у фоновому режимі.

**Увімкнення:**
1. Перейдіть до **Налаштування → Сповіщення → Push-сповіщення**
2. Натисніть **Увімкнути push-сповіщення**
3. Браузер запитує дозвіл — натисніть **Дозволити**
4. Натисніть **Перевірити сповіщення**

:::info Тільки в браузері, де ви це увімкнули
Push-сповіщення прив'язані до конкретного браузера та пристрою. Увімкніть на кожному пристрої, де хочете отримувати сповіщення.
:::

## Вибір подій для сповіщень

Після налаштування каналу сповіщень ви можете вибрати, які саме події спричинятимуть сповіщення:

**У Налаштування → Сповіщення → Події:**

| Подія | Рекомендовано |
|-------|--------------|
| Друк завершено | Так |
| Друк не вдався / скасований | Так |
| Print Guard: виявлено спагеті | Так |
| Помилка HMS (критична) | Так |
| Попередження HMS | Необов'язково |
| Низький рівень філаменту | Так |
| Помилка AMS | Так |
| Принтер відключено | Необов'язково |
| Нагадування про обслуговування | Необов'язково |
| Нічне резервне копіювання завершено | Ні (надокучливо) |

## Тихі години (без сповіщень вночі)

Уникайте пробудження через завершений друк о 03:00:

1. Перейдіть до **Налаштування → Сповіщення → Тихі години**
2. Увімкніть **Тихі години**
3. Встановіть час "від" і "до" (наприклад, **22:00 до 07:00**)
4. Виберіть, які події все ще можуть сповіщати в тихий період:
   - **Критичні помилки HMS** — рекомендовано залишити увімкненими
   - **Print Guard** — рекомендовано залишити увімкненими
   - **Друк завершено** — можна вимкнути вночі

:::tip Нічні друки без порушення спокою
Запускайте друки вночі з увімкненими тихими годинами. Print Guard стежить — а вранці ви отримаєте підсумок.
:::
