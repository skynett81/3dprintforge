# Віддалений сервер (Remote Nodes)

Функція віддалених серверів дозволяє з'єднувати кілька екземплярів Bambu Dashboard, щоб ви могли переглядати та керувати всіма принтерами з одного центрального інтерфейсу — незалежно від того, чи вони знаходяться в одній мережі або в різних місцях.

Перейдіть до: **https://localhost:3443/#settings** → **Інтеграції → Віддалені сервери**

## Сценарії використання

- **Вдома + офіс** — Переглядайте принтери в обох місцях з одного дашборду
- **Makerspace** — Центральний дашборд для всіх екземплярів у приміщенні
- **Гостьові екземпляри** — Надайте обмежений доступ клієнтам без повного доступу

## Архітектура

```
Основний екземпляр (ваш ПК)
  ├── Принтер A (локальний MQTT)
  ├── Принтер B (локальний MQTT)
  └── Віддалений сервер: Вторинний екземпляр
        ├── Принтер C (MQTT у віддаленому місці)
        └── Принтер D (MQTT у віддаленому місці)
```

Основний екземпляр опитує віддалені сервери через REST API та агрегує дані локально.

## Додавання віддаленого сервера

### Крок 1: Згенеруйте API-ключ на віддаленому екземплярі

1. Увійдіть до віддаленого екземпляра (наприклад, `https://192.168.2.50:3443`)
2. Перейдіть до **Налаштування → API-ключі**
3. Натисніть **Новий ключ** → дайте назву «Основний вузол»
4. Встановіть дозволи: **Читання** (мінімум) або **Читання + Запис** (для дистанційного керування)
5. Скопіюйте ключ

### Крок 2: Підключіться з основного екземпляра

1. Перейдіть до **Налаштування → Віддалені сервери**
2. Натисніть **Додати віддалений сервер**
3. Заповніть:
   - **Назва**: наприклад «Офіс» або «Гараж»
   - **URL**: `https://192.168.2.50:3443` або зовнішній URL
   - **API-ключ**: ключ з кроку 1
4. Натисніть **Тест з'єднання**
5. Натисніть **Зберегти**

:::warning Самопідписаний сертифікат
Якщо віддалений екземпляр використовує самопідписаний сертифікат, увімкніть **Ігнорувати TLS-помилки** — але робіть це лише для підключень у внутрішній мережі.
:::

## Агрегований вигляд

Після підключення віддалені принтери відображаються в:

- **Огляді парку** — позначені назвою віддаленого сервера та значком хмари
- **Статистиці** — агрегованій для всіх екземплярів
- **Сховищі філаменту** — зведеному огляді

## Дистанційне керування

З дозволом **Читання + Запис** ви можете керувати віддаленими принтерами безпосередньо:

- Пауза / Відновлення / Зупинка
- Додати до черги друку (завдання надсилається на віддалений екземпляр)
- Переглянути потік камери (проксований через віддалений екземпляр)

:::info Затримка
Потік камери через віддалений сервер може мати помітну затримку залежно від швидкості мережі та відстані.
:::

## Контроль доступу

Обмежте, якими даними ділиться віддалений сервер:

1. На віддаленому екземплярі: перейдіть до **Налаштування → API-ключі → [Назва ключа]**
2. Обмежте доступ:
   - Лише конкретні принтери
   - Без потоку камери
   - Тільки читання

## Стан та моніторинг

Стан кожного віддаленого сервера відображається в **Налаштування → Віддалені сервери**:

- **Підключено** — останнє опитування успішне
- **Відключено** — неможливо зв'язатися з віддаленим сервером
- **Помилка автентифікації** — API-ключ недійсний або прострочений
- **Остання синхронізація** — позначка часу останньої успішної синхронізації даних
