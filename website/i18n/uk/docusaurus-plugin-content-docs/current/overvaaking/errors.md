# Журнал помилок

Журнал помилок збирає всі помилки та HMS-попередження (Health, Maintenance, Safety) від ваших принтерів. Bambu Dashboard має вбудовану базу даних з 269+ HMS-кодами для принтерів Bambu Lab.

Перейдіть до: **https://localhost:3443/#errors**

## HMS-коди

Принтери Bambu Lab надсилають HMS-коди через MQTT, коли щось не так. Bambu Dashboard автоматично перекладає їх у зрозумілі повідомлення про помилки:

| Код | Приклад | Категорія |
|-----|---------|----------|
| `0700 0100 0001 0001` | Nozzle heatbreak clogged | Сопло/екструдер |
| `0700 0200 0002 0001` | AMS filament stuck | AMS |
| `0700 0300 0003 0001` | Bed leveling failed | Платформа друку |
| `0700 0500 0001 0001` | MC disconnect | Електроніка |

Повний список охоплює всі 269+ відомих кодів для X1C, X1C Combo, X1E, P1S, P1S Combo, P1P, P2S, P2S Combo, A1, A1 Combo, A1 mini, H2S, H2D та H2C.

## Ступінь серйозності

Помилки класифікуються за чотирма рівнями:

| Рівень | Колір | Опис |
|--------|-------|------|
| **Критичний** | Червоний | Вимагає негайних дій — друк зупинено |
| **Високий** | Помаранчевий | Слід розглянути швидко — друк може продовжитись |
| **Середній** | Жовтий | Слід перевірити — безпосередньої небезпеки немає |
| **Інфо** | Синій | Інформаційне повідомлення, дії не потрібні |

## Пошук та фільтрація

Використовуйте панель інструментів вгорі журналу помилок:

1. **Повнотекстовий пошук** — шукайте в повідомленні про помилку, HMS-коді або описі принтера
2. **Принтер** — показувати помилки лише від одного принтера
3. **Категорія** — AMS / Сопло / Пластина / Електроніка / Калібрування / Інше
4. **Серйозність** — Всі / Критичний / Високий / Середній / Інфо
5. **Дата** — фільтрувати за діапазоном дат
6. **Непідтверджені** — показувати лише непідтверджені помилки

Натисніть **Скинути фільтр** для перегляду всіх помилок.

## Посилання на вікі

Для кожного HMS-коду відображається посилання на Bambu Lab Wiki з:

- Повним описом помилки
- Можливими причинами
- Покроковими інструкціями з усунення несправностей
- Офіційними рекомендаціями Bambu Lab

Натисніть **Відкрити вікі** на записі помилки для відкриття відповідної сторінки вікі в новій вкладці.

:::tip Локальна копія
Bambu Dashboard кешує вміст вікі локально для офлайн-використання. Вміст оновлюється автоматично щотижня.
:::

## Підтвердження помилок

Підтвердження позначає помилку як оброблену, не видаляючи її:

1. Натисніть на помилку в списку
2. Натисніть **Підтвердити** (іконка галочки)
3. Введіть необов'язкову нотатку про те, що було зроблено
4. Помилка позначається галочкою та переміщується до списку «Підтверджені»

### Масове підтвердження

1. Виберіть кілька помилок за допомогою прапорців
2. Натисніть **Підтвердити вибрані**
3. Всі вибрані помилки підтверджуються одночасно

## Статистика

Вгорі журналу помилок відображається:

- Загальна кількість помилок за останні 30 днів
- Кількість непідтверджених помилок
- Найчастіший HMS-код
- Принтер з найбільшою кількістю помилок

## Експорт

1. Натисніть **Експортувати** (іконка завантаження)
2. Виберіть формат: **CSV** або **JSON**
3. Фільтр застосовується до експорту — спочатку встановіть потрібний фільтр
4. Файл завантажується автоматично

## Сповіщення про нові помилки

Активуйте сповіщення для нових HMS-помилок:

1. Перейдіть до **Налаштування → Сповіщення**
2. Позначте **Нові HMS-помилки**
3. Виберіть мінімальний ступінь серйозності для сповіщення (рекомендовано: **Високий** і вище)
4. Виберіть канал сповіщення

Дивіться [Сповіщення](../funksjoner/notifications) для налаштування каналів.
