# Автентифікація

Bambu Dashboard підтримує кількох користувачів з рольовим контролем доступу, API-ключами та необов'язковою двофакторною автентифікацією (2FA) через TOTP.

Перейдіть до: **https://localhost:3443/#settings** → **Користувачі та доступ**

## Користувачі

### Створення користувача

1. Перейдіть до **Налаштування → Користувачі**
2. Натисніть **Новий користувач**
3. Заповніть:
   - **Ім'я користувача** — використовується для входу
   - **Електронна пошта**
   - **Пароль** — рекомендується мінімум 12 символів
   - **Роль** — дивіться ролі нижче
4. Натисніть **Створити**

Новий користувач тепер може увійти на **https://localhost:3443/login**.

### Зміна пароля

1. Перейдіть до **Профіль** (верхній правий кут → натисніть на ім'я користувача)
2. Натисніть **Змінити пароль**
3. Введіть поточний пароль та новий пароль
4. Натисніть **Зберегти**

Адміністратори можуть скидати паролі інших з **Налаштування → Користувачі → [Користувач] → Скинути пароль**.

## Ролі

| Роль | Опис |
|---|---|
| **Адміністратор** | Повний доступ — всі налаштування, користувачі та функції |
| **Оператор** | Керування принтерами, перегляд всього, але без зміни системних налаштувань |
| **Гість** | Тільки читання — перегляд дашборду, історії та статистики |
| **API-користувач** | Тільки доступ через API — без веб-інтерфейсу |

### Власні ролі

1. Перейдіть до **Налаштування → Ролі**
2. Натисніть **Нова роль**
3. Виберіть дозволи окремо:
   - Перегляд дашборду / історії / статистики
   - Керування принтерами (пауза/зупинка/старт)
   - Управління сховищем філаменту
   - Управління чергою
   - Перегляд потоку камери
   - Зміна налаштувань
   - Управління користувачами
4. Натисніть **Зберегти**

## API-ключі

API-ключі надають програмний доступ без входу в систему.

### Створення API-ключа

1. Перейдіть до **Налаштування → API-ключі**
2. Натисніть **Новий ключ**
3. Заповніть:
   - **Назва** — описова назва (наприклад, «Home Assistant», «Python-скрипт»)
   - **Термін дії** — необов'язково, встановіть для безпеки
   - **Дозволи** — виберіть роль або конкретні дозволи
4. Натисніть **Згенерувати**
5. **Скопіюйте ключ зараз** — він відображається лише один раз

### Використання API-ключа

Додайте до HTTP-заголовку для всіх API-запитів:
```
Authorization: Bearer ВАШ_API_КЛЮЧ
```

Дивіться [API-пісочниця](../tools/playground) для тестування.

:::danger Безпечне зберігання
API-ключі мають той самий доступ, що й користувач, до якого вони прив'язані. Зберігайте їх безпечно та регулярно змінюйте.
:::

## TOTP 2FA

Увімкніть двофакторну автентифікацію за допомогою додатку-аутентифікатора (Google Authenticator, Authy, Bitwarden тощо):

### Увімкнення 2FA

1. Перейдіть до **Профіль → Безпека → Двофакторна автентифікація**
2. Натисніть **Увімкнути 2FA**
3. Відскануйте QR-код додатком-аутентифікатором
4. Введіть згенерований 6-значний код для підтвердження
5. Збережіть **коди відновлення** (10 одноразових кодів) у безпечному місці
6. Натисніть **Активувати**

### Вхід з 2FA

1. Введіть ім'я користувача та пароль як зазвичай
2. Введіть 6-значний TOTP-код з додатку
3. Натисніть **Увійти**

### Примусова 2FA для всіх користувачів

Адміністратори можуть вимагати 2FA для всіх користувачів:

1. Перейдіть до **Налаштування → Безпека → Примусова 2FA**
2. Увімкніть налаштування
3. Користувачі без 2FA будуть змушені налаштувати її при наступному вході

## Управління сесіями

- Стандартна тривалість сесії: 24 години
- Відрегулюйте в **Налаштування → Безпека → Тривалість сесії**
- Переглядайте активні сесії кожного користувача та завершуйте окремі сесії
