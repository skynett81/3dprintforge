# Резервне копіювання

3DPrintForge може створювати резервні копії всієї конфігурації, історії та даних, щоб ви могли легко відновити їх при збої системи, переміщенні сервера або проблемах оновлення.

Перейдіть до: **https://localhost:3443/#settings** → **Система → Резервне копіювання**

## Що включається до резервної копії

| Тип даних | Включено | Примітка |
|---|---|---|
| Налаштування та конфігурації принтерів | ✅ | |
| Історія друку | ✅ | |
| Сховище філаменту | ✅ | |
| Користувачі та ролі | ✅ | Паролі зберігаються у хешованому вигляді |
| Налаштування | ✅ | Включно з конфігураціями сповіщень |
| Журнал обслуговування | ✅ | |
| Проекти та рахунки | ✅ | |
| Бібліотека файлів (метадані) | ✅ | |
| Бібліотека файлів (файли) | Необов'язково | Може бути великою |
| Відео таймлапс | Необов'язково | Може бути дуже великою |
| Зображення галереї | Необов'язково | |

## Створення резервної копії вручну

1. Перейдіть до **Налаштування → Резервне копіювання**
2. Виберіть, що включити (дивіться таблицю вище)
3. Натисніть **Створити резервну копію зараз**
4. Відображається індикатор прогресу під час створення резервної копії
5. Натисніть **Завантажити** після завершення

Резервна копія зберігається як `.zip`-файл з позначкою часу в назві:
```
3dprintforge-backup-2026-03-22T14-30-00.zip
```

## Завантаження резервної копії

Файли резервних копій зберігаються в папці резервних копій на сервері (налаштовується). Крім того, їх можна завантажити безпосередньо:

1. Перейдіть до **Резервне копіювання → Існуючі резервні копії**
2. Знайдіть резервну копію у списку (сортується за датою)
3. Натисніть **Завантажити** (значок завантаження)

:::info Папка зберігання
Стандартна папка зберігання: `./data/backups/`. Змініть у **Налаштування → Резервне копіювання → Папка зберігання**.
:::

## Заплановане автоматичне резервне копіювання

1. Увімкніть **Автоматичне резервне копіювання** в **Резервне копіювання → Планування**
2. Виберіть інтервал:
   - **Щоденно** — запускається о 03:00 (налаштовується)
   - **Щотижнево** — в конкретний день та час
   - **Щомісяця** — перший день місяця
3. Виберіть **Кількість резервних копій для зберігання** (наприклад, 7 — старіші видаляються автоматично)
4. Натисніть **Зберегти**

:::tip Зовнішнє зберігання
Для важливих даних: змонтуйте зовнішній диск або мережевий диск як папку зберігання резервних копій. Тоді резервні копії збережуться навіть у разі відмови системного диска.
:::

## Відновлення з резервної копії

:::warning Відновлення перезаписує існуючі дані
Відновлення замінює всі існуючі дані вмістом файлу резервної копії. Переконайтеся, що у вас є свіжа резервна копія поточних даних.
:::

### З існуючої резервної копії на сервері

1. Перейдіть до **Резервне копіювання → Існуючі резервні копії**
2. Знайдіть резервну копію у списку
3. Натисніть **Відновити**
4. Підтвердіть у діалозі
5. Система автоматично перезапускається після відновлення

### З завантаженого файлу резервної копії

1. Натисніть **Завантажити резервну копію**
2. Виберіть `.zip`-файл з вашого комп'ютера
3. Файл перевіряється — ви бачите, що включено
4. Натисніть **Відновити з файлу**
5. Підтвердіть у діалозі

## Перевірка резервної копії

3DPrintForge перевіряє всі файли резервних копій перед відновленням:

- Перевіряє правильність ZIP-формату
- Переконується, що схема бази даних сумісна з поточною версією
- Показує попередження, якщо резервна копія з більш старої версії (міграція виконується автоматично)
