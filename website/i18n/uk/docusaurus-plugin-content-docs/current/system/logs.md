# Журнал сервера

Журнал сервера дає вам уявлення про те, що відбувається всередині 3DPrintForge — корисно для усунення несправностей, моніторингу та діагностики.

Перейдіть до: **https://localhost:3443/#logs**

## Вигляд у реальному часі

Потік журналу оновлюється в реальному часі через WebSocket:

1. Перейдіть до **Система → Журнал сервера**
2. Нові рядки журналу відображаються автоматично внизу
3. Натисніть **Зафіксувати внизу** для завжди прокрутки до останнього запису
4. Натисніть **Заморозити** для зупинки автопрокрутки та читання існуючих рядків

Стандартний вигляд показує останні 500 рядків журналу.

## Рівні журналу

Кожен рядок журналу має рівень:

| Рівень | Колір | Опис |
|---|---|---|
| **ERROR** | Червоний | Помилки, що впливають на функціональність |
| **WARN** | Помаранчевий | Попередження — щось може піти не так |
| **INFO** | Синій | Нормальна операційна інформація |
| **DEBUG** | Сірий | Детальна інформація для розробників |

:::info Конфігурація рівня журналу
Змініть рівень журналу в **Налаштування → Система → Рівень журналу**. Для нормальної роботи використовуйте **INFO**. Використовуйте **DEBUG** лише при усуненні несправностей, оскільки генерує набагато більше даних.
:::

## Фільтрація

Використовуйте панель фільтрів вгорі вигляду журналу:

1. **Рівень журналу** — показати лише ERROR / WARN / INFO / DEBUG або комбінацію
2. **Модуль** — фільтрувати за системним модулем:
   - `mqtt` — MQTT-зв'язок з принтерами
   - `api` — API-запити
   - `db` — операції бази даних
   - `auth` — події автентифікації
   - `queue` — події черги друку
   - `guard` — події Print Guard
   - `backup` — операції резервного копіювання
3. **Довільний текст** — пошук у тексті журналу (підтримує regex)
4. **Час** — фільтрувати за діапазоном дат

Комбінуйте фільтри для точного усунення несправностей.

## Типові ситуації помилок

### Проблеми MQTT-з'єднання

Шукайте рядки журналу від модуля `mqtt`:

```
ERROR [mqtt] Підключення до принтера XXXX не вдалось: Connection refused
```

**Рішення:** Перевірте, що принтер увімкнений, код доступу правильний та мережа працює.

### Помилки бази даних

```
ERROR [db] Міграція v95 не вдалась: SQLITE_CONSTRAINT
```

**Рішення:** Зробіть резервну копію та запустіть відновлення бази даних через **Налаштування → Система → Відновити базу даних**.

### Помилки автентифікації

```
WARN [auth] Невдала спроба входу для користувача admin з IP 192.168.1.x
```

Багато невдалих спроб входу може свідчити про атаку перебору. Перевірте, чи варто активувати список дозволених IP.

## Експорт журналів

1. Натисніть **Експортувати журнал**
2. Виберіть часовий период (стандарт: останні 24 години)
3. Виберіть формат: **TXT** (зрозуміло для людини) або **JSON** (машинозчитуваний)
4. Файл завантажується

Експортовані журнали корисні при повідомленні про помилки або зверненні до підтримки.

## Ротація журналів

Журнали ротуються автоматично:

| Налаштування | Стандарт |
|---|---|
| Макс. розмір файлу журналу | 50 МБ |
| Кількість ротованих файлів для зберігання | 5 |
| Загальний макс. розмір журналу | 250 МБ |

Відрегулюйте в **Налаштування → Система → Ротація журналів**. Старіші файли журналів автоматично стискаються за допомогою gzip.

## Розташування файлів журналу

Файли журналу зберігаються на сервері:

```
./data/logs/
├── 3dprintforge.log          (активний журнал)
├── 3dprintforge.log.1.gz     (ротований)
├── 3dprintforge.log.2.gz     (ротований)
└── ...
```

:::tip SSH-доступ
Для читання журналів безпосередньо на сервері через SSH:
```bash
tail -f ./data/logs/3dprintforge.log
```
:::
