# API-пісочниця

API-пісочниця дозволяє досліджувати та тестувати всі 177 API-ендпоінтів Bambu Dashboard безпосередньо в браузері — без написання коду.

Перейдіть до: **https://localhost:3443/api/docs**

## Що таке API-пісочниця?

Пісочниця — це інтерактивна версія OpenAPI-документації (Swagger UI), повністю інтегрована з дашбордом. Ви вже автентифіковані при вході в систему, тому можете тестувати ендпоінти безпосередньо.

## Навігація в документації

Ендпоінти організовані за категоріями:

| Категорія | Кількість ендпоінтів | Опис |
|---|---|---|
| Принтери | 24 | Отримати статус, керувати, налаштовувати |
| Друки / Історія | 18 | Отримати, шукати, експортувати історію |
| Філамент | 22 | Сховище, котушки, профілі |
| Черга | 12 | Управляти чергою друку |
| Статистика | 15 | Агрегована статистика та експорт |
| Сповіщення | 8 | Налаштувати та тестувати канали сповіщень |
| Користувачі | 10 | Користувачі, ролі, API-ключі |
| Налаштування | 14 | Читати та змінювати конфігурацію |
| Обслуговування | 12 | Завдання обслуговування та журнал |
| Інтеграції | 18 | HA, Tibber, вебхуки тощо |
| Бібліотека файлів | 14 | Завантажити, аналізувати, управляти |
| Система | 10 | Резервне копіювання, стан, журнал |

Натисніть на категорію для розгортання та перегляду всіх ендпоінтів.

## Тестування ендпоінту

1. Натисніть на ендпоінт (наприклад, `GET /api/printers`)
2. Натисніть **Try it out** (спробуйте)
3. Заповніть будь-які параметри (фільтри, пагінацію, ID принтера тощо)
4. Натисніть **Execute**
5. Перегляньте відповідь нижче: HTTP-статус-код, заголовки та JSON-тіло

### Приклад: Отримати всі принтери

```
GET /api/printers
```
Повертає список усіх зареєстрованих принтерів з поточним статусом у реальному часі.

### Приклад: Надіслати команду принтеру

```
POST /api/printers/{id}/command
Body: {"command": "pause"}
```

:::warning Виробниче середовище
API-пісочниця підключена до реальної системи. Команди надсилаються на справжні принтери. Будьте обережні з деструктивними операціями, такими як `DELETE` та `POST /command`.
:::

## Автентифікація

### Автентифікація сесії (залогований користувач)
Коли ви залогінені в дашборді, пісочниця вже автентифікована через сесійний cookie. Додаткова конфігурація не потрібна.

### Автентифікація API-ключем

Для зовнішнього доступу:

1. Натисніть **Authorize** (значок замку вгорі пісочниці)
2. Введіть ваш API-ключ у поле **ApiKeyAuth**: `Bearer ВАШ_КЛЮЧ`
3. Натисніть **Authorize**

Генеруйте API-ключі в **Налаштування → API-ключі** (дивіться [Автентифікація](../system/auth)).

## Обмеження запитів

API має обмеження **200 запитів за хвилину** на користувача/ключ. Пісочниця показує кількість запитів, що залишилися, у заголовку відповіді `X-RateLimit-Remaining`.

:::info Специфікація OpenAPI
Завантажте повну специфікацію OpenAPI у форматі YAML або JSON:
- `https://localhost:3443/api/docs/openapi.yaml`
- `https://localhost:3443/api/docs/openapi.json`

Використовуйте специфікацію для генерації клієнтських бібліотек на Python, TypeScript, Go тощо.
:::

## Тестування вебхуків

Тестуйте інтеграції вебхуків безпосередньо:

1. Перейдіть до `POST /api/webhooks/test`
2. Виберіть тип події зі списку
3. Система надсилає тестову подію на налаштовану URL вебхуку
4. Перегляньте запит/відповідь у пісочниці
