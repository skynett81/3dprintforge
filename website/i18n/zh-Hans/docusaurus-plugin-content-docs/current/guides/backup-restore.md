# 备份与恢复

Bambu Dashboard在本地存储所有数据——打印历史记录、耗材库存、设置、用户等。定期备份确保在服务器故障或迁移时不会丢失任何数据。

## 备份包含哪些内容？

| 数据 | 包含 | 备注 |
|------|------|------|
| 打印历史记录 | 是 | 所有日志和统计数据 |
| 耗材库存 | 是 | 线轴、重量、品牌 |
| 设置 | 是 | 所有系统设置 |
| 打印机配置 | 是 | IP地址、访问码 |
| 用户和角色 | 是 | 密码以哈希形式存储 |
| 通知配置 | 是 | Telegram令牌等 |
| 摄像头图像 | 可选 | 可能成为大文件 |
| 延时视频 | 可选 | 默认排除 |

## 自动夜间备份

默认情况下，每晚03:00运行自动备份。

**查看和配置自动备份：**
1. 进入**系统 → 备份**
2. 在**自动备份**下查看：
   - 上次成功备份和时间
   - 下次计划备份
   - 保存的备份数量（默认：7天）

**配置：**
- **时间** — 从默认03:00更改为合适的时间
- **保留时间** — 保留备份的天数（7、14、30天）
- **存储位置** — 本地文件夹（默认）或外部路径
- **压缩** — 默认启用（将大小减少60〜80%）

:::info 备份文件默认保存在这里
```
/path/to/bambu-dashboard/data/backups/
backup-2025-03-22-030000.tar.gz
backup-2025-03-21-030000.tar.gz
...
```
:::

## 手动备份

随时进行备份：

1. 进入**系统 → 备份**
2. 点击**立即备份**
3. 等待状态显示**已完成**
4. 点击**下载**下载备份文件

**通过终端备选方式：**
```bash
cd /sti/til/bambu-dashboard
node scripts/backup.js
```

备份文件以时间戳为文件名保存在`data/backups/`中。

## 从备份恢复

:::warning 恢复会覆盖现有数据
所有现有数据将被备份文件的内容替换。确保恢复到正确的文件。
:::

### 通过仪表板

1. 进入**系统 → 备份**
2. 点击**恢复**
3. 从列表中选择备份文件，或从磁盘上传备份文件
4. 点击**立即恢复**
5. 恢复后仪表板自动重启

### 通过终端

```bash
cd /sti/til/bambu-dashboard
node scripts/restore.js data/backups/backup-2025-03-22-030000.tar.gz
```

恢复后重启仪表板：
```bash
sudo systemctl restart bambu-dashboard
# 或
npm start
```

## 导出和导入设置

只想保存设置（而不是所有历史记录）？

**导出：**
1. 进入**系统 → 设置 → 导出**
2. 选择要包含的内容：
   - 打印机配置
   - 通知配置
   - 用户账号
   - 耗材品牌和配置文件
3. 点击**导出** — 下载一个`.json`文件

**导入：**
1. 进入**系统 → 设置 → 导入**
2. 上传`.json`文件
3. 选择要导入的部分
4. 点击**导入**

:::tip 对新安装很有用
导出的设置便于带到新服务器。新安装后导入，无需重新配置所有内容。
:::

## 迁移到新服务器

将Bambu Dashboard及所有数据迁移到新机器的方法：

### 第1步 — 在旧服务器上进行备份

1. 进入**系统 → 备份 → 立即备份**
2. 下载备份文件
3. 将文件复制到新服务器（USB、scp、网络共享）

### 第2步 — 在新服务器上安装

```bash
git clone https://github.com/skynett81/bambu-dashboard.git
cd bambu-dashboard
./install.sh
```

按照安装指南操作。你不需要配置任何内容——只需让仪表板运行起来。

### 第3步 — 恢复备份

仪表板在新服务器上运行后：

1. 进入**系统 → 备份 → 恢复**
2. 上传旧服务器的备份文件
3. 点击**立即恢复**

历史记录、耗材库存、设置和用户现在全部就位。

### 第4步 — 验证连接

1. 进入**设置 → 打印机**
2. 测试与每台打印机的连接
3. 检查IP地址是否仍然正确（新服务器可能有不同的IP）

## 良好备份习惯的建议

- **测试恢复** — 进行备份并在测试机器上至少恢复一次。未经测试的备份不算备份。
- **外部存储** — 定期将备份文件复制到外部磁盘或云存储（Nextcloud、Google Drive等）
- **设置警报** — 在**设置 → 通知 → 事件**下为"备份失败"启用通知，这样出问题时立即知道
