# 设置通知

Bambu Dashboard可以通知你从打印完成到严重错误的一切情况——通过Telegram、Discord、邮件或浏览器推送通知。

## 通知渠道概览

| 渠道 | 最适合 | 需要 |
|------|--------|------|
| Telegram | 快速，随处可用 | Telegram账号 + 机器人令牌 |
| Discord | 团队/社区 | Discord服务器 + Webhook URL |
| 邮件 (SMTP) | 正式通知 | SMTP服务器 |
| 浏览器推送 | 桌面通知 | 支持推送的浏览器 |

---

## Telegram机器人

### 第1步 — 创建机器人

1. 打开Telegram，搜索**@BotFather**
2. 发送`/newbot`
3. 给机器人命名（例如"Bambu 通知"）
4. 给机器人设置用户名（例如`bambu_notification_bot`）— 必须以`bot`结尾
5. BotFather回复一个**API令牌**：`123456789:ABCdefGHIjklMNOpqrsTUVwxyz`
6. 复制并保存此令牌

### 第2步 — 找到你的Chat ID

1. 与你的机器人开始对话（搜索用户名并点击**开始**）
2. 向机器人发送一条消息（例如"你好"）
3. 在浏览器中访问`https://api.telegram.org/bot<你的令牌>/getUpdates`
4. 找到`"chat":{"id": 123456789}` — 这是你的Chat ID

### 第3步 — 连接到仪表板

1. 进入**设置 → 通知 → Telegram**
2. 粘贴**机器人令牌**
3. 粘贴**Chat ID**
4. 点击**测试通知** — 你应该在Telegram中收到测试消息
5. 点击**保存**

:::tip 群组通知
想通知整个群组？将机器人添加到Telegram群组，找到群组Chat ID（负数，例如`-100123456789`）并替代使用。
:::

## Discord Webhook

### 第1步 — 在Discord中创建Webhook

1. 进入你的Discord服务器
2. 右键点击要接收通知的频道 → **编辑频道**
3. 进入**集成 → Webhooks**
4. 点击**新建Webhook**
5. 给它命名（例如"Bambu Dashboard"）
6. 选择头像（可选）
7. 点击**复制Webhook URL**

URL格式如下：
```
https://discord.com/api/webhooks/123456789/abcdefghijk...
```

### 第2步 — 输入到仪表板

1. 进入**设置 → 通知 → Discord**
2. 粘贴**Webhook URL**
3. 点击**测试通知** — Discord频道应该收到测试消息
4. 点击**保存**

## 邮件 (SMTP)

### 所需信息

你需要邮件提供商的SMTP设置：

| 提供商 | SMTP服务器 | 端口 | 加密 |
|-------|-----------|------|------|
| Gmail | smtp.gmail.com | 587 | TLS |
| Outlook/Hotmail | smtp-mail.outlook.com | 587 | TLS |
| Yahoo | smtp.mail.yahoo.com | 587 | TLS |
| 自有域名 | smtp.yourdomain.com | 587 | TLS |

:::warning Gmail需要应用密码
Gmail会阻止使用普通密码登录。你需要在Google账号 → 安全 → 两步验证 → 应用密码中创建**应用密码**。
:::

### 在仪表板中配置

1. 进入**设置 → 通知 → 邮件**
2. 填写：
   - **SMTP服务器**：例如`smtp.gmail.com`
   - **端口**：`587`
   - **用户名**：你的邮箱地址
   - **密码**：应用密码或普通密码
   - **发件人地址**：发送通知的邮箱
   - **收件人地址**：接收通知的邮箱
3. 点击**测试邮件**
4. 点击**保存**

## 浏览器推送通知

推送通知以系统通知形式显示在桌面上——即使浏览器标签在后台也会显示。

**启用：**
1. 进入**设置 → 通知 → 推送通知**
2. 点击**启用推送通知**
3. 浏览器请求权限 — 点击**允许**
4. 点击**测试通知**

:::info 仅在启用时所在的浏览器中有效
推送通知绑定到特定浏览器和设备。在每个想接收通知的设备上启用。
:::

## 选择要通知的事件

设置通知渠道后，你可以精确选择哪些事件触发通知：

**在设置 → 通知 → 事件下：**

| 事件 | 推荐 |
|------|------|
| 打印完成 | 是 |
| 打印失败 / 取消 | 是 |
| Print Guard：检测到意大利面 | 是 |
| HMS错误（严重） | 是 |
| HMS警告 | 可选 |
| 耗材不足 | 是 |
| AMS错误 | 是 |
| 打印机断开 | 可选 |
| 维护提醒 | 可选 |
| 夜间备份完成 | 否（太扰人） |

## 免打扰时段（夜间不通知）

避免凌晨3点被打印完成消息叫醒：

1. 进入**设置 → 通知 → 免打扰时段**
2. 启用**免打扰时段**
3. 设置开始和结束时间（例如**22:00到07:00**）
4. 选择在免打扰期间仍然通知的事件：
   - **严重HMS错误** — 建议保持开启
   - **Print Guard** — 建议保持开启
   - **打印完成** — 夜间可以关闭

:::tip 夜间打印不受打扰
开启免打扰时段进行夜间打印。Print Guard负责监控——你早上会收到摘要。
:::
