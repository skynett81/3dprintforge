# 远程服务器（Remote Nodes）

远程服务器功能允许您将多个 3DPrintForge 实例连接在一起，从一个统一的界面查看和控制所有打印机——无论它们位于同一网络还是不同地点。

访问地址：**https://localhost:3443/#settings** → **集成 → 远程服务器**

## 使用场景

- **家庭 + 办公室** — 从同一仪表板查看两处的打印机
- **创客空间** — 为房间内所有实例提供集中仪表板
- **访客实例** — 为客户提供有限访问权限，而不给予完整访问

## 架构

```
主实例（您的 PC）
  ├── 打印机 A（本地 MQTT）
  ├── 打印机 B（本地 MQTT）
  └── 远程服务器：从实例
        ├── 打印机 C（远程位置的 MQTT）
        └── 打印机 D（远程位置的 MQTT）
```

主实例通过 REST API 轮询远程服务器，并在本地聚合数据。

## 添加远程服务器

### 步骤 1：在远程实例上生成 API 密钥

1. 登录远程实例（例如 `https://192.168.2.50:3443`）
2. 进入**设置 → API 密钥**
3. 点击**新建密钥** → 命名为「主节点」
4. 设置权限：**读取**（最低要求）或**读取 + 写入**（用于远程控制）
5. 复制密钥

### 步骤 2：从主实例连接

1. 进入**设置 → 远程服务器**
2. 点击**添加远程服务器**
3. 填写：
   - **名称**：例如「办公室」或「车库」
   - **URL**：`https://192.168.2.50:3443` 或外部 URL
   - **API 密钥**：步骤 1 中获取的密钥
4. 点击**测试连接**
5. 点击**保存**

:::warning 自签名证书
如果远程实例使用自签名证书，请启用**忽略 TLS 错误**——但仅限于内部网络连接时使用此选项。
:::

## 聚合视图

连接后，远程打印机将显示在：

- **机群概览** — 标注远程服务器名称和云图标
- **统计数据** — 跨所有实例的聚合统计
- **耗材库存** — 汇总概览

## 远程控制

拥有**读取 + 写入**权限时，可直接控制远程打印机：

- 暂停 / 继续 / 停止
- 添加到打印队列（任务发送到远程实例）
- 查看摄像头画面（通过远程实例代理）

:::info 延迟说明
通过远程服务器的摄像头画面可能会有明显延迟，具体取决于网络速度和物理距离。
:::

## 访问控制

限制远程服务器共享的数据范围：

1. 在远程实例上：进入**设置 → API 密钥 → [密钥名称]**
2. 限制访问范围：
   - 仅特定打印机
   - 禁用摄像头画面
   - 只读模式（仅允许读取）

## 健康状态与监控

每个远程服务器的状态显示在**设置 → 远程服务器**中：

- **已连接** — 最近一次轮询成功
- **已断开** — 无法访问远程服务器
- **身份验证失败** — API 密钥无效或已过期
- **最后同步** — 最近一次成功数据同步的时间戳
