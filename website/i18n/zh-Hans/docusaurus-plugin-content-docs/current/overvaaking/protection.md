# 打印守护

打印守护是 Bambu Dashboard 的实时监控系统。它持续监控摄像头、传感器和打印机数据，并在出现问题时执行可配置的操作。

访问地址：**https://localhost:3443/#protection**

## XCam 事件检测

拓竹打印机在 AI 摄像头检测到问题时，会通过 MQTT 发送 XCam 事件：

| 事件 | 代码 | 严重程度 |
|---|---|---|
| 检测到意面 | `xcam_spaghetti` | 危急 |
| 打印平台脱落 | `xcam_detach` | 高 |
| 首层异常 | `xcam_first_layer` | 高 |
| 拉丝 | `xcam_stringing` | 中 |
| 挤出错误 | `xcam_extrusion` | 高 |

对于每种事件类型，您可以配置一个或多个操作：

- **通知** — 通过已激活的通知渠道发送提醒
- **暂停** — 暂停打印以便手动检查
- **停止** — 立即取消打印
- **忽略** — 忽略该事件（仍会记录日志）

:::danger 默认行为
默认情况下，XCam 事件设置为**通知**和**暂停**。如果您完全信任 AI 检测结果，可改为**停止**。
:::

## 传感器监控

打印守护持续监控传感器数据，并在出现异常时报警：

### 温度偏差

1. 进入**打印守护 → 温度**
2. 设置**与目标温度的最大偏差**（建议：喷嘴 ±5°C，热床 ±3°C）
3. 选择**偏差时的操作**：通知 / 暂停 / 停止
4. 设置**延迟时间**（秒），在执行操作前给温度稳定的时间

### 耗材不足

系统计算线卷上的剩余耗材量：

1. 进入**打印守护 → 耗材**
2. 设置**最低限制**（克），例如 50 g
3. 选择操作：**暂停并通知**（推荐），以便手动更换线卷

### 打印停止检测

检测打印何时意外停止（MQTT 超时、断丝等）：

1. 启用**停止检测**
2. 设置**超时时间**（建议：无数据 120 秒 = 已停止）
3. 操作：始终通知——打印机可能已经停止

## 配置

### 启用打印守护

1. 进入**设置 → 打印守护**
2. 开启**启用打印守护**
3. 选择要监控的打印机
4. 点击**保存**

### 按打印机配置规则

不同打印机可以有不同的规则：

1. 在打印守护概览中点击某台打印机
2. 关闭**继承全局规则**
3. 为该打印机单独配置规则

## 日志与事件历史

所有打印守护事件均被记录：

- 进入**打印守护 → 日志**
- 按打印机、事件类型、日期和严重程度筛选
- 点击某条事件查看详细信息以及执行的操作
- 将日志导出为 CSV

:::tip 误报处理
如果打印守护触发了不必要的暂停，请在**打印守护 → 设置 → 灵敏度**中调整灵敏度。从「低」开始，逐步提高。
:::
