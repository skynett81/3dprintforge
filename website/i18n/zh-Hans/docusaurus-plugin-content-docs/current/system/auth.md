# 身份验证

3DPrintForge 支持多用户访问，具备基于角色的访问控制、API 密钥以及可选的 TOTP 双因素验证（2FA）。

访问地址：**https://localhost:3443/#settings** → **用户与访问**

## 用户

### 创建用户

1. 进入**设置 → 用户**
2. 点击**新建用户**
3. 填写：
   - **用户名** — 用于登录
   - **电子邮件地址**
   - **密码** — 建议至少 12 位字符
   - **角色** — 见下方角色说明
4. 点击**创建**

新用户现在可以在 **https://localhost:3443/login** 登录。

### 修改密码

1. 进入**个人资料**（右上角 → 点击用户名）
2. 点击**修改密码**
3. 填写当前密码和新密码
4. 点击**保存**

管理员可在**设置 → 用户 → [用户名] → 重置密码**中重置其他用户的密码。

## 角色

| 角色 | 说明 |
|---|---|
| **管理员** | 完全访问权限——所有设置、用户和功能 |
| **操作员** | 控制打印机、查看所有内容，但不能修改系统设置 |
| **访客** | 只读——查看仪表板、历史记录和统计数据 |
| **API 用户** | 仅限 API 访问——无 Web 界面 |

### 自定义角色

1. 进入**设置 → 角色**
2. 点击**新建角色**
3. 单独选择权限：
   - 查看仪表板 / 历史记录 / 统计数据
   - 控制打印机（暂停/停止/开始）
   - 管理耗材库存
   - 管理打印队列
   - 查看摄像头画面
   - 修改设置
   - 管理用户
4. 点击**保存**

## API 密钥

API 密钥提供无需登录的程序化访问方式。

### 创建 API 密钥

1. 进入**设置 → API 密钥**
2. 点击**新建密钥**
3. 填写：
   - **名称** — 描述性名称（例如「Home Assistant」、「Python 脚本」）
   - **过期日期** — 可选，为安全起见建议设置
   - **权限** — 选择角色或具体权限
4. 点击**生成**
5. **立即复制密钥** — 密钥仅显示一次

### 使用 API 密钥

在所有 API 调用的 HTTP 请求头中添加：
```
Authorization: Bearer 您的API密钥
```

参阅 [API 试验场](../tools/playground) 进行测试。

:::danger 安全存储
API 密钥与关联用户具有相同的访问权限。请妥善保管，并定期轮换。
:::

## TOTP 双因素验证

使用验证器应用（Google Authenticator、Authy、Bitwarden 等）启用双因素验证：

### 启用 2FA

1. 进入**个人资料 → 安全 → 双因素验证**
2. 点击**启用 2FA**
3. 用验证器应用扫描二维码
4. 输入生成的 6 位验证码进行确认
5. 将**恢复代码**（10 个一次性代码）保存在安全位置
6. 点击**激活**

### 使用 2FA 登录

1. 像往常一样输入用户名和密码
2. 输入应用中的 6 位 TOTP 验证码
3. 点击**登录**

### 强制所有用户启用 2FA

管理员可以要求所有用户启用 2FA：

1. 进入**设置 → 安全 → 强制 2FA**
2. 启用该设置
3. 未启用 2FA 的用户在下次登录时将被强制设置

## 会话管理

- 默认会话有效期：24 小时
- 在**设置 → 安全 → 会话时长**中调整
- 查看每位用户的活跃会话并终止单个会话
