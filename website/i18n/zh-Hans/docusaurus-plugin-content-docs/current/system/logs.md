# 服务器日志

服务器日志让您深入了解 Bambu Dashboard 内部发生的情况——对故障排查、监控和诊断非常有用。

访问地址：**https://localhost:3443/#logs**

## 实时查看

日志流通过 WebSocket 实时更新：

1. 进入**系统 → 服务器日志**
2. 新日志行自动显示在底部
3. 点击**锁定底部**以始终滚动到最新日志
4. 点击**冻结**暂停自动滚动，便于阅读现有内容

默认视图显示最近 500 条日志。

## 日志级别

每条日志都有对应级别：

| 级别 | 颜色 | 说明 |
|---|---|---|
| **ERROR** | 红色 | 影响功能的错误 |
| **WARN** | 橙色 | 警告——某些功能可能出现问题 |
| **INFO** | 蓝色 | 正常运行信息 |
| **DEBUG** | 灰色 | 详细的开发人员信息 |

:::info 日志级别配置
在**设置 → 系统 → 日志级别**中更改日志级别。正常运行时使用 **INFO**。仅在排查问题时使用 **DEBUG**，因为它会生成更多数据。
:::

## 筛选

使用日志视图顶部的筛选工具栏：

1. **日志级别** — 仅显示 ERROR / WARN / INFO / DEBUG 或组合
2. **模块** — 按系统模块筛选：
   - `mqtt` — 与打印机的 MQTT 通信
   - `api` — API 请求
   - `db` — 数据库操作
   - `auth` — 认证事件
   - `queue` — 打印队列事件
   - `guard` — 打印守护事件
   - `backup` — 备份操作
3. **全文搜索** — 在日志文本中搜索（支持正则表达式）
4. **时间** — 按日期范围筛选

组合筛选条件可进行精确故障排查。

## 常见错误情况

### MQTT 连接问题

查找 `mqtt` 模块的日志：

```
ERROR [mqtt] 连接打印机 XXXX 失败：Connection refused
```

**解决方法：** 检查打印机是否开机，访问码是否正确，网络是否正常。

### 数据库错误

```
ERROR [db] 迁移 v95 失败：SQLITE_CONSTRAINT
```

**解决方法：** 先备份，然后通过**设置 → 系统 → 修复数据库**运行数据库修复。

### 认证错误

```
WARN [auth] 用户 admin 从 IP 192.168.1.x 登录失败
```

大量登录失败可能表示暴力破解攻击。考虑是否应启用 IP 白名单。

## 导出日志

1. 点击**导出日志**
2. 选择时间范围（默认：最近 24 小时）
3. 选择格式：**TXT**（人类可读）或 **JSON**（机器可读）
4. 文件将自动下载

导出的日志在提交 Bug 报告或联系技术支持时非常有用。

## 日志轮转

日志自动轮转：

| 设置 | 默认值 |
|---|---|
| 单个日志文件最大大小 | 50 MB |
| 保留的轮转文件数量 | 5 |
| 日志总最大大小 | 250 MB |

在**设置 → 系统 → 日志轮转**中调整。旧日志文件自动以 gzip 压缩。

## 日志文件位置

日志文件存储在服务器上：

```
./data/logs/
├── bambu-dashboard.log          （活动日志）
├── bambu-dashboard.log.1.gz     （已轮转）
├── bambu-dashboard.log.2.gz     （已轮转）
└── ...
```

:::tip SSH 访问
通过 SSH 直接在服务器上读取日志：
```bash
tail -f ./data/logs/bambu-dashboard.log
```
:::
