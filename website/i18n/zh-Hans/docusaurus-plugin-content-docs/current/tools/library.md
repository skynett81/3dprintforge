# 文件库

文件库是存储和管理所有 3D 模型和 G-code 文件的中央位置——支持自动 G-code 分析，并与 MakerWorld 和 Printables 集成。

访问地址：**https://localhost:3443/#library**

## 上传模型

### 单文件上传

1. 进入**文件库**
2. 点击**上传**或将文件拖放到上传区域
3. 支持的格式：`.3mf`、`.gcode`、`.bgcode`、`.stl`、`.obj`
4. 上传后文件自动进行分析

:::info 存储位置
文件存储在**设置 → 文件库 → 存储位置**中配置的目录。默认：`./data/library/`
:::

### 批量上传

将整个文件夹拖放进来，一次性上传所有支持的文件。文件在后台处理，完成后系统发送通知。

## G-code 分析

上传后，`.gcode` 和 `.bgcode` 文件将自动进行分析：

| 指标 | 说明 |
|---|---|
| 预计打印时间 | 根据 G-code 命令计算的时间 |
| 耗材消耗量 | 每种材料/颜色的克数和米数 |
| 层数 | 总层数 |
| 层厚 | 检测到的层厚 |
| 材料 | 检测到的材料（PLA、PETG 等） |
| 填充密度 | 如元数据中有提供 |
| 支撑材料 | 预计支撑材料重量 |
| 打印机型号 | 元数据中的目标打印机 |

分析数据显示在文件卡片中，并供[费用估算器](../analytics/costestimator)使用。

## 文件卡片与元数据

每个文件卡片显示：
- **文件名**和格式
- **上传日期**
- **缩略图**（来自 `.3mf` 或自动生成）
- **分析后的打印时间**和耗材消耗量
- **标签**和类别
- **关联打印** — 打印过的次数

点击卡片打开包含完整元数据和历史记录的详情视图。

## 组织管理

### 标签

添加标签以便快速搜索：
1. 点击文件 → **编辑元数据**
2. 输入标签（逗号分隔）：`benchy, 测试, PLA, 校准`
3. 使用标签筛选器在文件库中搜索

### 类别

将文件整理到类别中：
- 在侧栏点击**新建类别**
- 将文件拖放到类别中
- 类别可以嵌套（支持子类别）

## 关联 MakerWorld

1. 进入**设置 → 集成 → MakerWorld**
2. 使用拓竹账号登录
3. 返回文件库：点击文件 → **关联到 MakerWorld**
4. 在 MakerWorld 上搜索模型，选择正确结果
5. 从 MakerWorld 导入元数据（设计者、许可、评分）

关联后，文件卡片上显示设计者名称和原始 URL。

## 关联 Printables

1. 进入**设置 → 集成 → Printables**
2. 粘贴您的 Printables API 密钥
3. 用与 MakerWorld 相同的方式将文件关联到 Printables 模型

## 发送到打印机

从文件库可直接发送到打印机：

1. 点击文件 → **发送到打印机**
2. 选择目标打印机
3. 选择 AMS 槽位（用于多色打印）
4. 点击**立即打印**或**加入队列**

:::warning 直接发送
直接发送将立即开始打印，无需在 Bambu Studio 中确认。请确认打印机已准备就绪。
:::
